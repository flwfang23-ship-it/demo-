# H5 Demo 分批建设流程

## 项目概述

**产品**：小程序SaaS营销平台 — 客户管理后台（商户自助端）H5 演示版
**参考文档**：`000大纲/A02b_客户后台（0521）.md`
**定位**：纯演示用，无后端逻辑，所有数据均为静态Mock
**技术方案**：纯 HTML + CSS + JS，无构建工具，浏览器直接打开即可预览
**视口**：PC端管理后台，最小宽度 1200px，左侧侧边栏 + 右侧内容区布局

---

## 页面范围（基于A02b）

根据 A02b 的页面树状图，客户管理后台包含以下模块：

| 编号 | 模块 | 核心页面 |
|------|------|----------|
| C01 | 首页概览（商户推广工作台） | 三大模块聚合页：②昨日数据（收入概览）+ ③财务数据（资金提现）+ ①取链模块（供应商Tab + 活动卡片网格） |
| C02 | 小程序管理 | 小程序列表 + 注册/编辑表单 + 小程序详情 + 订阅开关管理 |
| C03 | 推广位管理 | 推广位列表 + 新增/编辑表单 + 设为默认 + 删除 |
| C04 | 权益页配置 | 左右分栏配置页 + Tab切换 + 6坑位网格 + 活动选择器弹窗 + 推广位管理弹窗 |
| C05 | 日报 | 日报列表 + 日报详情 |
| C06 | 账号设置 | 企业信息 + 修改密码 + 操作日志 |

### 第1批：设计规范 + PC布局框架 + 侧边栏导航

**目标**：建立整个管理后台的视觉语言和页面骨架，后续所有批次都基于此。

**产出文件**：

| 文件 | 用途 |
|------|------|
| `css/variables.css` | CSS变量（颜色/字体/间距/圆角/阴影/动画） |
| `css/common.css` | 全局重置、排版、按钮/表格/表单/卡片等公共组件基类 |
| `css/layout.css` | 侧边栏（240px）+ 顶栏（56px）+ 内容区布局 |
| `js/nav.js` | 左侧菜单展开/收起、菜单点击切换、面包屑更新 |
| `js/mock-data.js` | 全局Mock数据（小程序/推广位/活动/日报/账号） |
| `index.html` | 主框架：侧边栏 + 顶栏 + 内容容器 + 页面占位 |

**关键约定（后续批次必须遵守）**：

- 主色：`#1677ff`，成功：`#52c41a`，警告：`#faad14`，危险：`#ff4d4f`
- 侧边栏宽度：240px（收起时 64px）
- 顶栏高度：56px
- 内容区最小宽度：`calc(100vw - 240px)`，背景 `#f5f5f5`
- 页面切换方式：侧边栏菜单点击 → JS 切换内容区 HTML
- 菜单结构（对应A02b页面树）：C01首页 / C02小程序管理 / C03推广位管理 / C04权益页配置 / C05日报 / C06账号设置
- 所有可交互元素（按钮/开关/弹窗/表单）需有 hover/active 态

### 第2批：C01 首页概览（商户推广工作台） + C02 小程序管理

**目标**：实现首页商户推广工作台和小程序管理完整流程。

**产出文件**：

| 文件 | 用途 |
|------|------|
| `pages/dashboard.html` | 首页推广工作台（三大模块：②昨日数据 + ③财务数据 + ①取链模块） |
| `pages/miniprogram-list.html` | 小程序列表（表格 + 搜索筛选 + 分页 + 订阅开关行内操作） |
| `pages/miniprogram-form.html` | 注册/编辑小程序表单（名称/AppID/类型/描述） |
| `pages/miniprogram-detail.html` | 小程序详情（基础信息 + 部署进度 + 运营数据摘要） |
| `css/dashboard.css` | 推广工作台样式（昨日数据卡片 + 财务卡片 + Tab栏 + 活动卡片网格 + 弹窗） |
| `css/miniprogram.css` | 小程序管理样式 |
| `js/dashboard.js` | 工作台交互（数据加载/提现弹窗/Tab切换/卡片渲染/取链弹窗） |
| `js/miniprogram.js` | 列表交互（搜索/筛选/分页/订阅开关切换/批量操作） |

**C01 仪表盘结构（商户推广工作台）**：

页面采用上下分区布局，共 3 个核心模块。详细规格见《G_商户推广工作台（0521）.md》。

**顶部区域（左右分栏）**：
- **左侧 2/3 — 模块② 昨日数据（收入概览区）**：白色卡片，"收入数据概览"标题 + 弱化提示文字。横向排列"昨日预估佣金(元)"和"昨日订单数"两个核心指标（32px Bold 大号字体），下方展示环比变化（如"+12.5%"）。点击卡片跳转日报详情页（C05-A-a）。
- **右侧 1/3 — 模块③ 财务数据（资金提现区）**：渐变蓝色强调色卡片。大号字体展示"账户余额(元)"+ 具体金额（36px Bold 白色）。[提现]按钮（白色背景+蓝色文字，高对比度）。底部并排展示"提现审核中(元)"和"已提现(元)"两个副指标（12px，70%透明度）。未绑定账户时显示"请先绑定提现账户"+ [去绑定]按钮。点击[提现]弹出提现申请弹窗（输入金额→校验→确认提交）。

**底部区域（全宽）**：
- **模块① 取链模块（业务推广核心区）**：顶部供应商Tab切换栏（全部 / 饿了么联盟 / 美团联盟），切换为前端本地过滤。下方活动卡片自适应网格（大屏3列），每张卡片包含：活动主图（16:9，无图时默认占位图）、活动主标题（16px Bold，最多2行省略）、活动副标题（13px #666，最多1行）、供应商标签（小圆角Tag，美团红色/饿了么蓝色）、预估佣金（16px Bold 品牌色高亮）、有效期（12px #999，"截止 YYYY-MM-DD"格式）、[立即推广]按钮（品牌色全宽圆角按钮）。点击[立即推广]弹出取链确认弹窗→选择推广位+链接类型→生成推广链接→展示结果（可复制/分享）。

**C02 小程序管理结构**：
- 顶部操作栏：搜索框 + 状态筛选 + 订阅状态筛选 + [注册小程序]按钮
- 表格列：小程序名称 / AppID / 类型 / 部署状态 / 订阅开关(行内Switch) / 注册时间 / 操作(详情/编辑)
- 批量操作：勾选框 + [批量开启] / [批量关闭]按钮
- 分页：20条/页
- 注册/编辑表单弹窗：名称 + AppID + 类型下拉 + 描述
- 详情页：基础信息卡片 + 部署进度时间线 + 运营数据摘要

**Mock数据**（写入 mock-data.js）：
- 至少8个小程序，覆盖不同状态（正常/审核中/已停用）和部署进度
- 订阅开关状态混合

### 第3批：C03 推广位管理 + C04 权益页配置（核心模块）

**目标**：实现推广位管理列表和权益页可视化配置，这是整个Demo最重要的交互模块。

**产出文件**：

| 文件 | 用途 |
|------|------|
| `pages/promotion-list.html` | 推广位列表（表格 + 新增/编辑弹窗） |
| `pages/benefit-config.html` | 权益页配置主页（左右分栏 + Tab + 6坑位网格） |
| `css/promotion.css` | 推广位管理样式 |
| `css/benefit-config.css` | 权益页配置样式（核心） |
| `components/modal.css` | 通用弹窗样式 |
| `components/toast.css` | Toast样式 |
| `js/promotion.js` | 推广位列表交互 |
| `js/benefit-config.js` | 权益页配置交互（Tab切换/坑位替换/弹窗） |
| `js/modal.js` | 弹窗组件（活动选择器D01 / 推广位管理D02 / 确认弹窗） |
| `js/toast.js` | Toast消息提示 |

**C03 推广位管理结构**：
- 表格列：推广位名称 / 推广位ID / 是否默认 / 绑定活动数 / 创建时间 / 操作
- 操作按钮：编辑 / 设为默认 / 删除（默认推广位删除按钮disabled）
- 新增/编辑：弹窗表单（名称输入框）
- 设为默认：二次确认后切换
- 删除：二次确认（默认推广位不可删）

**C04 权益页配置结构（核心，按A02b规格）**：
- **左侧（240px）**：页面Tab列表（垂直排列），默认主页始终置顶 + 自定义页，末尾 [⚙️ 推广位管理] 按钮
- **右侧主内容区**：
  - 状态栏：当前页面启用状态标签 + 推广参数 `?promoId=t_618` + [📋 复制]按钮
  - 6坑位网格（3×2）：活动卡片（主图占位 + 活动名称 + 跳转类型标签）
  - 坑位卡片点击 → 弹出 D01 活动选择器
- **D01 活动选择器弹窗**：
  - 搜索框（300ms防抖模拟）
  - 活动列表（单选，已选高亮，含跳转类型标签）
  - 点击目标活动 → 即时替换坑位 + Toast提示 + 弹窗关闭
- **D02 推广位管理弹窗**：
  - 页面Checkbox列表（勾选=启用，默认主页disabled锁定）
  - [+新建推广位]按钮 → 输入名称 → 创建
- **交互联动**：
  - Tab切换 → 状态栏刷新 + 6坑位刷新
  - 替换活动 → 2次点击完成，无需保存按钮
  - 新建推广位 → Tab列表自动新增 + 自动选中

**Mock数据**（写入 mock-data.js）：
- 至少3个推广位（1个默认 + 2个自定义）
- 至少3个推广页面Tab（1个默认主页 + 2个自定义页）
- 每个页面6个活动坑位，活动有不同跳转类型
- 至少15个可选活动用于活动选择器

### 第4批：C05 日报 + C06 账号设置

**目标**：实现日报查看和账号设置模块。

**产出文件**：

| 文件 | 用途 |
|------|------|

**C05 日报结构**：

- 详情页：当日汇总卡片（收入/订单数/推广位数量）+ 各推广位明细表格

- 三个Tab：企业信息 / 修改密码 / 操作日志

**Mock数据**（写入 mock-data.js）：
- 至少15条日报记录
- 每条日报对应2-3个推广位明细
- 至少20条操作日志
