## 功能 B07：新人指引

---

### 1. 需求背景与目标 (Why)

新注册商户登录后面对功能完整的管理后台，不清楚从何入手。系统需要通过分步引导，让商户明确知道"还要做什么才能开始赚钱"，降低上手门槛，提升首单转化率。

核心目标：
- **进度可视化**：商户一眼看到自己完成了哪些步骤，还差几步
- **分步引导**：每步告知需要做什么、需要准备什么、操作入口在哪
- **降低流失**：已完成注册但未上传执照/未注册小程序的商户，通过首页横幅持续触达

---

#### 2.2 步骤状态流转图

```
          ┌──────────┐
          │   todo   │  未解锁，灰色 🔒 图标
          └────┬─────┘
               │ 上一步完成（done）
               ▼
          ┌──────────┐
          │ current  │  当前步骤，白色圆形蓝边框 + 光晕 + 橙色"进行中"标签
          └────┬─────┘
               │ 用户完成该步骤操作
               ▼
          ┌──────────┐
          │   done   │  已完成，蓝色渐变圆形 ✓ + 蓝色"已完成"标签
          └──────────┘
```

**状态说明表**：

| 状态名称 | 状态含义 | 进入条件 | 视觉表现 |
|----------|----------|----------|----------|
| done | 步骤已完成 | 用户完成了该步骤对应的操作 | 蓝色渐变圆形 + ✓ + 蓝色标签"已完成" |
| current | 当前需要操作的步骤 | 上一步为 done | 白色圆形蓝边框 + 外圈光晕 + 橙色标签"进行中" |
| todo | 尚未解锁的步骤 | 前面还有未完成的步骤 | 灰色圆形 + 🔒 + 灰色标签"未完成" |

**步骤解锁规则**：

| 规则编号 | 规则描述 |
|----------|----------|
| RULE-ONB-001 | Step1 注册成功后自动为 done |
| RULE-ONB-002 | Step2 上传营业执照后自动为 done，Step3 解锁为 current |
| RULE-ONB-003 | Step3 注册至少1个小程序成功后自动为 done，Step4 解锁为 current |
| RULE-ONB-004 | Step4 用户点击"前往推广位管理"或"我知道了"后为 done |
| RULE-ONB-005 | 所有步骤 done 后，首页引导横幅消失 |
