## 1. 需求背景与目标 (Why)

商户在管理后台选择联盟活动后，需要获取真实的推广链接用于投放。这些链接不是 SaaS 平台自己生成的——必须通过推广位预置的平台参数（mediaId / appkey / appSecret / sid / pid 等），调用美团联盟或饿了么联盟的官方取链接口，由平台侧返回。

**商户不需要、也不应该看到** 8 个 platformConfig 字段。这些字段由运营在总后台配置（或推广位创建时后端自动同步），取链时由后端自动读取并传入平台接口。

## 2. 平台返回数据格式（🔑 核心）

### 2.1 美团 — `meitundata.referralLinkMap`

美团返回一个扁平的数字 key → 链接值映射：

```json
{
  "meitundata": {
    "referralLinkMap": {
      "1": "https://click.meituan.com/t?t=1&c=2&p=FTNY7bxz1fzy",
      "2": "http://dpurl.cn/QXwMH4Nz",
      "3": "imeituan://www.meituan.com/takeout/browser?inner_url=https%3A%2F%2Fclick.meituan.com%2F...",
      "4": "/index/pages/h5/h5?weburl=https%3A%2F%2Fclick.meituan.com%2F...",
      "5": "1来美团，吃得更好，生活更好❤️复制整条信息，打开👉美团👈 http://...",
      "6": "http://p0.meituan.net/marketingcpsmedia/aedabb7999619ff04e61979595541b4b20684.jpg"
    }
  }
}
```

| 数字 key | 含义 | 说明 |
|----------|------|------|
| `1` | 短链接 | 短 URL，如 `dpurl.cn/xxx` |
| `2` | 长链接 | 完整追踪链接，如 `click.meituan.com/...` |
| `3` | 唤起协议 (deeplink) | `imeituan://` scheme，唤起美团App |
| `4` | 小程序路径 | 美团小程序内页面路径 |
| `5` | 团口令 | 文案口令，含 emoji，可直接复制发送 |
| `6` | 小程序二维码 | 二维码图片 URL |

> **特点**：扁平结构，6 个固定 key，全部为字符串值。前端按 1→6 顺序逐行展示即可。

### 2.2 饿了么 — `elemedata.link`

饿了么返回一个**嵌套分组结构**。`link` 对象内按推广渠道分为多个子对象，每个子对象内包含该渠道的具体链接字段：

```json
{
  "elemedata": {
    "link": {
      "alipay_promotion": {},
      "app_promotion": {},
      "h5_promotion": {},
      "wx_promotion": {},
      "taobao_promotion": {
        "h5_url": "https://market.m.taobao.com/app/starlink/wakeup-transit/pages/download?...",
        "img_url": "https://union.elemecdn.com/public/mediaShare/image/202605268bde...jpg",
        "scheme_url": "tbopen://m.taobao.com/tbopen/index.html?&action=ali.open.nav&...",
        "tb_qr_code": "https://union.elemecdn.com/public/mediaShare/qrcode/20260526687f...jpg",
        "tb_watchword": "￥nNqo5DUaUHe￥/ HU7405",
        "tb_watchword_suggest": "67 HU7405  2:/£nNqo5DUaUHe《三闪购大额满减红包"
      },
      "full_taobao_word": "67 HU7405  2:/£nNqo5DUaUHe《三闪购大额满减红包",
      "taobao_word": "￥nNqo5DUaUHe￥/ HU7405",
      "tb_mini_qrcode": "https://union.elemecdn.com/public/mediaShare/qrcode/20260526687f...jpg",
      "tb_qr_code": "https://union.elemecdn.com/public/mediaShare/image/202605268bde...jpg",
      "tb_scheme_url": "tbopen://m.taobao.com/tbopen/index.html?..."
    }
  }
}
```

**分组 + 字段映射表**：

| 分组 (link 子对象) | 包含字段 | 前端展示策略 |
|-------------------|----------|-------------|
| `taobao_promotion` | `h5_url` — H5推广页<br>`scheme_url` — 唤起协议<br>`img_url` — 推广图片<br>`tb_qr_code` — 二维码<br>`tb_watchword` — 淘宝口令(短)<br>`tb_watchword_suggest` — 淘宝口令(建议版) | **核心展示区**。有值即展开渲染 |
| `h5_promotion` | (字段名待定，由饿了么实际返回决定) | 非空时展示 |
| `app_promotion` | (字段名待定) | 非空时展示 |
| `alipay_promotion` | (字段名待定) | 非空时展示 |
| `wx_promotion` | (字段名待定) | 非空时展示 |
| **顶层扁平字段** | `full_taobao_word` — 完整淘宝口令<br>`taobao_word` — 淘宝口令(短)<br>`tb_mini_qrcode` — 淘宝小程序二维码<br>`tb_qr_code` — 淘宝二维码图片<br>`tb_scheme_url` — 淘宝唤起协议 | 作为独立行展示 |

> **特点**：嵌套分组结构，每组是一个推广渠道。大部分内容集中在 `taobao_promotion` 组。**空对象 `{}` 的分组不展示**。顶层扁平字段与分组内字段可能有重复（如 `taobao_word` 与 `taobao_promotion.tb_watchword`），前端按"有则展示"处理。

### 3.2 后端数据流（🔗 核心链路）

```
  ┌──────────────────────────────────────────────────────────────────┐
  │                       后端取链流程                                 │
  │                                                                  │
  │  ① 前端请求: POST /merchant/api/promotion/generate-link           │
  │     参数: { activityId, promotionSlotId }                        │
  │                                                                  │
  │  ② 后端查活动信息 → supplier + activity_id(第三方ID)               │
  │     查推广位 platformConfig（8字段，仅后端可见）                     │
  │                                                                  │
  │  ③ 根据 supplier 路由:                                            │
  │     ├─ meituan → 美团联盟取链接口                                  │
  │     │   传入: meituanMediaId, meituanSlotSid, meituanAppkey,      │
  │     │         meituanAppSecret, activity_id                       │
  │     │   返回: meitundata.referralLinkMap (数字key→链接值)          │
  │     │                                                             │
  │     └─ eleme  → 饿了么联盟取链接口                                  │
  │         传入: elemeMediaId, elemePid, taobaoAppkey,               │
  │               taobaoAppSecret, activity_id                        │
  │         返回: elemedata.link (嵌套分组结构)                         │
  │                                                                  │
  │  ④ 后端将平台返回的数据结构透传给前端                                │
  │                                                                  │
  └──────────────────────────────────────────────────────────────────┘
```

| 状态名称 | 进入条件 | 前端表现 |
|----------|----------|----------|

### 3.4 取链结果回写取链清单（B05 → B05-C 联动）

商户在推广链接抽屉点击[生成推广链接]后，后端除将结果返回前端展示外，还需**同步将真实链接数据回写至取链清单**（promoChainList），确保取链清单中始终保存最新一次取链的真实结果。

#### 3.4.1 触发时机

| 触发场景 | 触发位置 | 说明 |
|----------|----------|------|
| 商户手动取链 | 推广链接抽屉 → [生成推广链接] | API-M05-01 返回成功后立即触发 |
| 静默入列 | 推广位创建成功后（API-M03-03 内部触发） | 见 [[B05-C_活动清单+预设活动（0526）]] |

> 两次触发走同一套字段映射，区别仅在于触发时机不同。

#### 3.4.2 回写策略

| 条件 | 处理 |
|------|------|
| 同 promoId + activityId 已存在 | **覆盖**该条记录的所有链接字段为最新值 |
| 同 promoId + activityId 不存在 | **新增**一条记录，source 标记为 `"manual"` |
| 取链失败（接口报错/超时） | **不回写**，取链清单保持旧值 |

#### 3.4.3 字段映射

后端从平台返回中提取字段，写入取链清单对应记录。

**美团**：

| 取链清单字段 | 美团返回来源 |
|-------------|-------------|
| `shortUrl` | `referralLinkMap["1"]` |
| `longUrl` | `referralLinkMap["2"]` |
| `deeplink` | `referralLinkMap["3"]` |
| `miniPath` | `referralLinkMap["4"]` |
| `groupToken` | `referralLinkMap["5"]` |
| `qrcode` | `referralLinkMap["6"]` |

**饿了么**：

| 取链清单字段 | 饿了么返回来源 |
|-------------|-------------|
| `longUrl` | `taobao_promotion.h5_url` |
| `deeplink` | `taobao_promotion.scheme_url` |
| `groupToken` | `taobao_promotion.tb_watchword` |
| `searchToken` | `taobao_promotion.tb_watchword_suggest` |
| `qrcode` | `taobao_promotion.tb_qr_code` |
| `promoImage` | `taobao_promotion.img_url` |
| `fullGroupToken` | 顶层 `full_taobao_word` 或 `taobao_word` |
| `miniQrcode` | 顶层 `tb_mini_qrcode` |
| `tbSchemeUrl` | 顶层 `tb_scheme_url` |

> 字段为空时写入空字符串，取链清单展示侧自行判断是否显示"—"。

## 6. 业务规则

| 规则编号 | 规则描述 | 触发条件 | 处理逻辑 |
|----------|----------|----------|----------|
| RULE-001 | platformConfig 仅后端可见 | 所有接口 | 前端不传不存不展示；后端从DB读取仅服务端使用 |
| RULE-002 | 按 supplier 路由平台 | 取链请求 | meituan→美团接口；eleme→饿了么接口 |
| RULE-003 | 美团扁平渲染 | supplier=meituan | 遍历 links 对象，key 1→6 按固定标签展示 |

| RULE-009 | 实时取链 | 每次点击[生成] | 不缓存，每次实时调平台接口 |

// ====== 改造后的 generateDrawerLink() ======
async function generateDrawerLink() {
  var slotId = document.getElementById('drawerPromoSelect').value;
  var act = currentDrawerActivity;
  if (!act || !slotId) return;

  setDrawerLoading(true);

  try {
    var resp = await fetch('/merchant/api/promotion/generate-link', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ activityId: act.id, promotionSlotId: slotId })
    });
    var json = await resp.json();

    if (json.code !== 0) {
      showDrawerError(json.message);
      return;
    }

    var supplier = json.data.supplier;
    var links = json.data.links || {};

    if (supplier === 'meituan') {
      renderMeituanLinks(links);
    } else if (supplier === 'eleme') {
      renderElemeLinks(links);
    } else {
      showDrawerError('未知供应商类型');
    }

  } catch (e) {
    showDrawerError('网络异常，请稍后重试');
  }
}

// ====== 美团：扁平列表渲染 ======
function renderMeituanLinks(links) {
  var labelMap = {
    '1': '短链接', '2': '长链接', '3': '唤起协议',
    '4': '小程序路径', '5': '团口令', '6': '小程序二维码'
  };

  var entries = [];
  for (var i = 1; i <= 6; i++) {
    var val = links[String(i)];
    if (val != null && val !== '') {
      entries.push({ key: String(i), label: labelMap[String(i)] || ('类型' + i), value: val });
    }
  }

  // 兜底：数字key之外的字段追加到末尾
  Object.keys(links).forEach(function(k) {
    if (!labelMap[k]) {
      entries.push({ key: k, label: k, value: links[k] });
    }
  });

  if (entries.length === 0) {
    showDrawerEmpty('该活动暂未返回推广链接，请联系运营');
    return;
  }

  var html = '<div class="drawer-links-list">';
  entries.forEach(function(e) {
    html += buildLinkRow(e.label, e.value);
  });
  html += '</div>';

  document.getElementById('drawerResultBox').className = 'drawer-result-box has-data';
  document.getElementById('drawerResultBox').innerHTML = html;
}

// ====== 饿了么：分组折叠面板渲染 ======
function renderElemeLinks(links) {
  var groupLabelMap = {
    'taobao_promotion': '淘宝推广', 'h5_promotion': 'H5推广',
    'app_promotion': 'App推广', 'alipay_promotion': '支付宝推广',
    'wx_promotion': '微信推广'
  };

  var fieldLabelMap = {
    'h5_url': 'H5推广页', 'scheme_url': '唤起协议', 'img_url': '推广图片',
    'tb_qr_code': '二维码', 'tb_watchword': '淘宝口令(短)',
    'tb_watchword_suggest': '淘宝口令(建议版)',
    'full_taobao_word': '完整淘宝口令', 'taobao_word': '淘宝口令(短)',
    'tb_mini_qrcode': '淘宝小程序二维码', 'tb_scheme_url': '淘宝唤起协议'
  };

  var html = '';

  // 先渲染分组（子对象）
  var groupKeys = Object.keys(groupLabelMap);
  groupKeys.forEach(function(gk) {
    var groupData = links[gk];
    var groupLabel = groupLabelMap[gk];
    var isEmpty = !groupData || Object.keys(groupData).length === 0;

    html += '<div class="drawer-link-group' + (isEmpty ? ' is-empty' : '') + '">';
    html += '<div class="drawer-link-group-header" onclick="toggleLinkGroup(this)">';
    html += '<span class="group-arrow">' + (isEmpty ? '▶' : '▼') + '</span>';
    html += '<span class="group-label">' + groupLabel + '</span>';
    if (isEmpty) html += '<span class="group-empty-hint">(无内容)</span>';
    html += '</div>';

    if (!isEmpty) {
      html += '<div class="drawer-link-group-body">';
      Object.keys(groupData).forEach(function(fk) {
        var fv = groupData[fk];
        if (fv != null && fv !== '') {
          html += buildLinkRow(fieldLabelMap[fk] || fk, fv);
        }
      });
      html += '</div>';
    }
    html += '</div>';
    delete links[gk]; // 标记已处理
  });

  // 渲染剩余顶层字段（"其他"区）
  var remaining = Object.keys(links).filter(function(k) {
    return links[k] != null && links[k] !== '' && typeof links[k] !== 'object';
  });
  if (remaining.length > 0) {
    html += '<div class="drawer-link-group">';
    html += '<div class="drawer-link-group-header" onclick="toggleLinkGroup(this)">';
    html += '<span class="group-arrow">▼</span>';
    html += '<span class="group-label">其他</span>';
    html += '</div>';
    html += '<div class="drawer-link-group-body">';
    remaining.forEach(function(k) {
      html += buildLinkRow(fieldLabelMap[k] || k, links[k]);
    });
    html += '</div></div>';
  }

  if (!html) {
    showDrawerEmpty('该活动暂未返回推广链接，请联系运营');
    return;
  }

  document.getElementById('drawerResultBox').className = 'drawer-result-box has-data';
  document.getElementById('drawerResultBox').innerHTML = html;
}

// ====== 构建单行（文本/图片） ======
function buildLinkRow(label, value) {
  var isImage = /(img_url|qr_code|qrcode|\.(jpg|png|gif|webp)(\?|$))/i.test(label)
             || (/^https?:\/\/.*\.(jpg|png|gif|webp)/i.test(value) && /qr|img|image/i.test(label));

  if (isImage) {
    return '<div class="drawer-link-row">' +
      '<span class="drawer-link-label">' + label + '</span>' +
      '<img src="' + value + '" class="drawer-link-img" onclick="openImageLightbox(\'' + value + '\')" ' +
      'onerror="this.outerHTML=\'<span class=drawer-link-value>\'+this.src+\'</span>\'">' +
      '<button class="btn-copy-link" onclick="copyLinkText(\'' + value.replace(/'/g, "\\'") + '\')">📋</button>' +
    '</div>';
  }

  var display = value.length > 80 ? value.substring(0, 80) + '...' : value;
  return '<div class="drawer-link-row">' +
    '<span class="drawer-link-label">' + label + '</span>' +
    '<span class="drawer-link-value" title="' + value.replace(/"/g, '&quot;') + '">' + display + '</span>' +
    '<button class="btn-copy-link" onclick="copyLinkText(\'' + value.replace(/'/g, "\\'") + '\')">📋</button>' +
  '</div>';
}

## 模块汇总

> 1. **platformConfig 仅后端可见**：8 个平台参数对商户端完全透明，取链时仅传 activityId + slotId。
> 2. **两种数据结构，两种渲染模式**：美团→扁平列表，饿了么→分组折叠面板。前端根据 `supplier` 字段判断。
> 3. **饿了么分组展示**：非空分组默认展开，空分组折叠+灰显+(无内容)。顶层扁平字段归入"其他"区。
> 4. **图片智能识别**：key 含 `img`/`qr_code`/`qrcode` 或 value 为图片 URL → 渲染缩略图+灯箱放大。
> 5. **去掉了文案素材单选组**：后端一次返回全部链接类型，不需要用户预选。
