# 活动管理：B端功能模块规格

> 注：本文档按 G 模板（B端）结构编写。运营在后台配置活动信息（含第三方活动ID），配置完成后商户端即可在活动列表看到已上架的活动。商户选择活动后，系统根据活动的供应商（美团/饿了么）调用对应平台的取链接口，结合推广位的平台参数生成推广链接。**第三方活动ID** 是本模块的核心字段，用于锁定第三方平台对应的活动，取链时必须传入。

---

## 模块概览

| 项目 | 内容 |
|------|------|
| 模块名称 | 活动管理 |
| 所属层级 | 两层皆有（🏢 总后台配置活动 + 🏪 客户端小程序取链） |
| 功能入口 | 🏢 总后台：左侧菜单 → 活动管理；🏪 客户端：小程序 → 权益页 → 活动列表 → 取链 |
| 覆盖功能 | 后台活动管理、商户端活动列表与取链 |
| 覆盖页面 | 🏢 总后台：B02-A 活动列表、B02-A-a 新增/编辑活动表单；🏪 客户端：小程序活动列表页、取链确认弹窗 |
| 时间 | 2026-05-26 |
| 依赖模块 | 推广位管理模块（取链时需选择推广位，推广位预配置了各平台参数）、客户管理模块（客户登录态校验） |

> **系统预置规则**：系统上线时无需预置活动数据，由运营按需新增。

> **术语约定**：
> - **联盟** = 业务概念，指美团联盟和饿了么联盟两大推广平台
> - **供应商（supplier）** = 活动表的字段名，枚举值 `meituan`（美团）、`eleme`（饿了么）
> - **第三方活动** = 京东/打车等其他平台的活动，属 [[B04_活动管理-第三方（0526）]] 范畴，不在本文档范围
> - **⚠️ 本文档为活动基础文档**：后续基于活动的新增字段、页面改动（如 B05-B 的 H5 模板关联、B05-C 的预设活动标记、B06 的口令自动更新过滤等）由各迭代文档承接，本文档不回溯更新。

## 后台活动管理

---

### 1. 需求背景与目标 (Why)

运营人员需要在后台统一配置和管理营销活动，作为商户端取链的数据源。通过后台配置活动的基础信息、供应商、佣金、第三方活动ID 等字段，控制商户端可见的活动范围和展示顺序。

---

### 2. 业务流程与逻辑 (How - 宏观)

#### 2.1 总后台操作路径（🏢）

> 平台运营人员在总后台配置和管理活动的完整操作路径。

```
  进入活动列表        筛选/搜索活动        点击[新增活动]       填写活动信息
  (后台→活动管理)  ───►  (B02-A列表)  ───►  (B02-A-a表单)  ───►  (保存)
       │                    │                    │                  │
       │                    │ 点击[编辑]         │ 校验失败         │ 保存成功
       ▼                    ▼                    ▼                  ▼
   (需管理员登录)      编辑活动表单           标红提示            返回列表
                            │
                            │ 点击[上架/下架]
                            ▼
                        状态切换（即时生效）
```

| 步骤 | 角色 | 触发动作 | 所在页面 | 前置条件 | 后续页面 |
|------|------|----------|----------|----------|----------|
| 1 | 🏢 运营 | 进入活动管理菜单 | B02-A 活动列表 | 管理员角色登录 | - |
| 2 | 🏢 运营 | 筛选/搜索活动 | B02-A 列表 | - | 列表刷新 |
| 3 | 🏢 运营 | 点击[新增活动] | B02-A 列表 | - | B02-A-a 新增表单 |
| 4 | 🏢 运营 | 填写活动信息并保存 | B02-A-a 表单 | 校验通过 | 返回列表，新增一行 |
| 5 | 🏢 运营 | 点击[编辑] | B02-A 列表 | 活动存在 | B02-A-a 编辑表单（回填数据） |
| 6 | 🏢 运营 | 修改信息并保存 | B02-A-a 表单 | 校验通过 | 返回列表，刷新该行 |
| 7 | 🏢 运营 | 点击[上架]/[下架] | B02-A 列表 | 活动存在 | 即时切换状态，商户端同步生效/失效 |

#### 2.3 状态流转图

```
  ┌──────────────┐   运营[新增活动]     ┌──────────────┐
  │   无活动      │──────────────────►│   已下架      │
  │   (初始态)   │   默认下架         │   (下架态)   │
  └──────────────┘                    └──────────────┘
                                           │
                                           │ 运营[上架]
                                           ▼
                                      ┌──────────────┐
                                      │   已上架      │
                                      │   (上架态)   │
                                      └──────────────┘
                                           │
                                           │ 运营[下架]
                                           ▼
                                      ┌──────────────┐
                                      │   已下架      │
                                      │   (下架态)   │
                                      └──────────────┘
```

**状态说明表**：

| 状态名称 | 状态含义 | 进入条件 | 可触达的状态 | 操作角色 |
|----------|----------|----------|-------------|---------|
| 已下架 | 活动不可在商户端展示 | 新增活动默认下架 / 运营手动下架 | 已上架 | 🏢 总后台运营 |
| 已上架 | 活动在商户端可见，商户可取链 | 运营点击[上架] | 已下架 | 🏢 总后台运营 |

> **说明**：活动状态仅控制商户端可见性。已上架的活动下架后，商户端不再展示该活动，但已生成的推广链接不失效。

### 5. 跨层通用----------------------------------------------------------------------------------------------------------------------

> 本功能以总后台为主，但活动数据会下发到商户端。以下标注适用层级。

#### 5.1 数据流（本功能相关）

| 数据项 | 来源 | 更新时机 | 数据范围（总后台） | 数据范围（客户端） | 缓存策略 |
|--------|------|----------|--------------------|--------------------|----------|
| 活动列表 | API-ACT-01 | 页面加载 / 搜索 / 翻页 | 全平台活动 | - | 无缓存，实时请求 |
| 活动详情 | API-ACT-04 | 编辑时进入表单页 | 全平台活动 | - | 无缓存 |

#### 5.2 接口规格（本功能相关）

##### 接口列表

| 接口ID | 接口名称 | 请求方式 | 接口路径 | 适用层级 | 调用页面 | 调用时机 |
|--------|----------|----------|----------|----------|----------|----------|
| API-ACT-01 | 获取活动列表 | GET | /api/admin/activities | 🏢 总后台 | B02-A | 页面加载/搜索/翻页 |
| API-ACT-02 | 新增/编辑活动 | POST | /api/admin/activity/save | 🏢 总后台 | B02-A-a | 运营点击保存 |
| API-ACT-03 | 更新活动状态 | PUT | /api/admin/activity/status | 🏢 总后台 | B02-A | 运营点击上架/下架 |
| API-ACT-04 | 获取活动详情 | GET | /api/admin/activity/detail | 🏢 总后台 | B02-A-a | 编辑时进入表单页 |

## 模块汇总

**关键设计决策**：

1. **第三方活动ID 是核心纽带**：每个活动有唯一的第三方活动ID（如美团活动ID、饿了么活动ID），取链时系统将此 ID 连同推广位的平台参数（APPKEY/SECRET等）传给第三方接口，生成带跟踪参数的推广链接。
2. **状态控制可见性**：活动仅有两个状态（上架/下架），商户端仅可见已上架的活动。下架后已生成的推广链接不失效。
3. **展示字段与逻辑字段分离**：佣金、有效期、会场推广规则均为展示用途，不影响实际取链逻辑。实际取链仅依赖：供应商 + 第三方活动ID + 推广位平台参数。
4. **排序控制客户端展示**：运营通过排序值控制商户端活动卡片的展示顺序，数字越大越靠前。
5. **推广位是取链的前置条件**：商户取链时必须选择推广位，推广位的平台参数（APPKEY/SECRET等）由运营在推广位详情页预配置。
