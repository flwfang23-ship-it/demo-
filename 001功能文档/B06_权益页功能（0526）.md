## 功能 A001：组件库管理（🏢 总后台）

---

### 1. 需求背景与目标 (Why)

权益页是商户面向C端用户展示权益活动的落地页。页面由多个"组件"拼装而成——标题组件用于分区，链接组件用于跳转到具体活动。

组件本身由开发人员预编译（写死在代码中），但哪些组件对商户可见、可点击组件跳转到哪个平台的哪个活动，需要运营在总后台统一管控。运营可随时上下架组件，灵活控制商户端可用组件范围，无需开发介入。

---

### 2. 业务流程与逻辑 (How - 宏观)

#### 2.1 总后台操作路径（🏢）

```
  进入组件库列表        筛选组件类型          点击[编辑]可点击组件
  (页面管理→组件库) ───► (全部/标题/链接Tab) ───► 编辑弹窗 D-B6-B
       │                    │                    │
       │                    │ 切换启用/禁用开关  │ 选活动（系统自动判定跳转方式）
       │                    ▼                    ▼
       │                 组件状态更新         活动过滤逻辑：
       │                                      显示三类活动（满足其一即可）：
       │                                      ① 有小程序路径的活动
       │                                      ② 自动更新+H5卡片的活动
       │                                      ③ 所有第三方活动
       │                                             │
       │                                             │ 选中活动
       │                                             ▼
       │                                      系统自动判定跳转方式：
       │                                      条件② → 跳H5
       │                                      条件①③ → 跳小程序
       │                                             │
       │                                             ▼
       │                                      保存 → Toast"组件配置已更新"
       │
       ▼
  (需管理员登录)
```

| 步骤 | 角色 | 触发动作 | 所在页面 | 前置条件 | 后续页面 |
|------|------|----------|----------|----------|----------|
| 1 | 🏢 运营 | 进入组件库列表 | 页面 B6-A | 管理员登录 | - |
| 2 | 🏢 运营 | 切换组件类型Tab | 页面 B6-A | - | 列表按类型过滤 |
| 3 | 🏢 运营 | 切换组件启用/禁用 | 页面 B6-A | 组件存在 | 组件状态即时生效，商户端同步更新 |
| 4 | 🏢 运营 | 编辑可点击组件 | 页面 B6-A | 组件类型=link | 弹窗 D-B6-B |

| 6 | 🏢 运营 | 保存配置 | 弹窗 D-B6-B | 活动已选 | Toast"组件配置已更新"，关闭弹窗 |

#### 2.2 组件分类与定义

```
  组件
  ├── 不可点击（type=title）
  │   ├── 一级标题（大标题，页面主标题）
  │   ├── 二级标题（模块标题，带图标+文案）
  │   └── 三级标题（子模块标题，纯文字）
  │
  └── 可点击（type=link）
      ├── 大卡片（大图+活动名+平台标签）
      ├── 小卡片（小图+活动名）
      └── 横幅（通栏banner）
```

| 组件分类 | type 值 | 可点击 | 终端行为 | 总后台可配置项 |
|----------|---------|--------|----------|---------------|
| 各级标题 | `title` | 否 | 仅展示，无点击响应 | 无（仅启用/禁用） |
| 跳转组件 | `link` | 是 | 点击跳转至 H5 页面 或 第三方小程序 | 跳转平台 + 目标活动 |

#### 2.3 可点击组件的跳转逻辑

活动可选条件（满足其一即可）：

| 条件编号 | 条件 | 跳转方式 |
|----------|------|----------|
| ① | 活动参数中配置了小程序路径（`miniProgramPath` 非空） | 通过微信开放标签 `<wx-open-launch-weapp>` 跳转至小程序，目标路径 = `miniProgramPath` |
| ② | 活动管理处开了「口令自动更新」（`autoRefreshToken = true`）+ 配置了 H5 卡片（H5 页面地址非空） | 跳转至 SaaS 平台 H5 活动页（携带活动口令） |
| ③ | 活动类型为「第三方活动」 | 通过微信开放标签 `<wx-open-launch-weapp>` 跳转至第三方小程序，目标路径 = 活动配置的 `miniProgramPath` |

> **跳转方式决策**：条件 ② 的活动 → 跳 H5；条件 ① 和 ③ 的活动 → 跳小程序。每个活动的跳转方式由活动自身属性决定，运营无需手动选择平台。

```
  用户点击权益页上的可点击组件
      │
      ├── 活动满足条件 ②（自动更新 + H5卡片）
      │   └── 跳转至 SaaS 平台 H5 活动页（携带活动口令）
      │
      ├── 活动满足条件 ①（有小程序路径）
      │   └── 通过微信开放标签 <wx-open-launch-weapp> 跳转至小程序
      │       └── 目标路径 = 活动配置的 miniProgramPath
      │
      └── 活动满足条件 ③（第三方活动）
          └── 通过微信开放标签 <wx-open-launch-weapp> 跳转至第三方小程序
              └── 目标路径 = 活动配置的 miniProgramPath
```

##### 3.2.4 活动选择过滤规则（核心逻辑）

活动下拉列表的候选活动满足以下三种条件之一即可，取**并集**：

**条件 ①：活动配置了小程序路径**

| 条件编号 | 条件 | 数据来源 | 说明 |
|----------|------|----------|------|
| FILTER-01 | 活动已配置小程序跳转路径 | 活动表 `miniProgramPath` 非空 | 确保点击后有可用的跳转目标路径 |

> **跳转方式**：满足条件 ① 的活动 → 通过微信开放标签跳转至小程序。

**条件 ②：活动开启了口令自动更新 + 配置了 H5 卡片**

| 条件编号 | 条件 | 数据来源 | 说明 |
|----------|------|----------|------|
| FILTER-02A | 活动已配置「口令自动更新」 | 活动表 `autoRefreshToken = true` | 确保该活动的口令由定时任务持续刷新，跳转后用户拿到的口令不过期 |
| FILTER-02B | 活动已配置 H5 卡片（H5 页面地址） | 活动表 `h5PageUrl` 非空 | 确保有可跳转的 H5 活动页 |

> 活动必须**同时满足** FILTER-02A + FILTER-02B。
> **跳转方式**：满足条件 ② 的活动 → 跳转至 SaaS 平台 H5 活动页（携带活动口令）。

**条件 ③：第三方活动（全部可选）**

| 条件编号 | 条件 | 数据来源 | 说明 |
|----------|------|----------|------|
| FILTER-03 | 活动类型为「第三方活动」 | 活动表 `activityType = 'thirdParty'` | 所有第三方活动（京东/打车/其他分组下的活动）均出现在下拉列表中，不校验口令自动更新和路径 |

> **跳转方式**：满足条件 ③ 的活动 → 通过微信开放标签跳转至第三方小程序。

> **过滤后端的实现**：接口 API-B06-003 返回活动列表时，后端做 UNION 查询：
> ```sql
> -- 条件 ①：活动配置了小程序路径
> SELECT * FROM activities
> WHERE miniProgramPath IS NOT NULL
>   AND miniProgramPath != ''
> UNION
> -- 条件 ②：口令自动更新 + H5 卡片
> SELECT * FROM activities
> WHERE auto_refresh_token = true
>   AND h5PageUrl IS NOT NULL
>   AND h5PageUrl != ''
> UNION
> -- 条件 ③：所有第三方活动
> SELECT * FROM activities
> WHERE activity_type = 'third_party'
> ```
> 前端直接使用返回数据渲染下拉选项，不额外过滤。

> **跳转方式由活动自身属性决定**：条件 ② 的活动默认跳 H5，条件 ① 和 ③ 的活动默认跳小程序。运营无需在弹窗中手动选择跳转平台，选中活动后系统自动确定跳转方式。
>
> **"其他活动不可选择"**：不满足上述任一条件的活动不出现在下拉列表中。若运营发现找不到想要的活动，需引导其先去活动管理页配置相关参数。

##### 3.2.5 跳转预览提示规则

选中活动后，系统根据活动满足的条件自动判定跳转方式，在弹窗底部「跳转预览」区域显示对应提示文案：

| 活动满足条件 | 跳转方式 | 提示文案模板 | 数据来源 |
|-------------|----------|-------------|----------|
| 条件 ②（自动更新 + H5卡片） | H5 | `跳转至「{activityH5Name}」（H5活动页）` | `{activityH5Name}` 取自选中活动的 H5 页面标题 |
| 条件 ①（有小程序路径） | 小程序 | `跳转至「{platformName}」平台下的小程序` | `{platformName}` 取自选中活动所属平台名称 |
| 条件 ③（第三方活动） | 小程序 | `跳转至「{platformName}」平台下的小程序` | `{platformName}` 取自选中活动所属第三方平台名称（如：京东、美团、饿了么） |

> 前端逻辑：`activityId` 变更时，根据活动数据中携带的 `jumpType` 字段（`h5` / `miniprogram`）实时更新底部的提示文案。无需运营手动选择平台。

## 功能 A003：默认样式配置（🏢 总后台）

---

### 1. 需求背景与目标 (Why)

新商户注册后，其权益页默认为空画布，商户需从零搭建页面，上手门槛高。运营需要在总后台预先配置一套"默认样式"——即一套预设的画布组件布局模板。新商户注册时，系统自动将该默认样式写入商户画布，使权益页自带一套可用的初始配置，商户在此基础上调整即可，降低使用门槛。

### 1. 需求背景与目标 (Why)

商户需要一个可视化的编辑器来搭建自己的权益页。商户不具备技术能力，因此编辑器采用**纯拖拽模式**——商户从左侧组件库中将已上线的组件拖入手机模拟器画布，调整排列顺序，保存后即可发布。商户不可编辑组件的内部逻辑（跳转地址、样式等均由总后台运营统一配置）。

## 模块汇总

### 关键设计决策

1. **三层职责分离**：开发预编译组件模板（HTML + CSS），总后台运营管理组件的启用/禁用和跳转配置，商户纯拖拽使用。各层只做自己权限范围内的事，不越界。

2. **组件库三分组**：客户端组件库按数据来源分为三大分组：装修组件（开发预编译的标题栏等）、联盟活动（来自活动管理）、第三方活动（来自第三方活动管理，按平台子分组）。商户从三个分组中拖拽组件到画布，统一编排。

3. **活动选择并集规则**：可点击组件的目标活动池取三种条件的并集：① 有小程序路径的活动 ② 开启口令自动更新且配置了 H5 卡片的活动 ③ 所有第三方活动。三种条件满足其一即可入选，跳转方式由活动自身属性自动判定。

4. **跳转方式自动判定**：H5 和第三方小程序在技术上是两种不同的跳转实现（H5 页面跳转 vs 微信开放标签），但运营无需手动选择跳转平台。选中活动后，系统根据活动满足的条件自动判定：条件 ② 的活动跳 H5，条件 ① 和 ③ 的活动跳小程序。减少运营配置负担，避免选错跳转方式。

5. **配置实时生效**：运营修改组件的跳转配置后，所有已使用该组件的商户权益页在C端展示时都会实时反映最新的跳转目标。这避免了"改了配置但已发布的页面不更新"的问题。

6. **组件禁用不强制移除**：运营禁用某组件后，已使用该组件的商户画布不受影响，避免运营操作造成商户侧"组件凭空消失"的困惑。仅阻止新拖入。

7. **默认样式降低上手门槛**：运营在总后台预配一套默认画布布局，新商户注册时自动应用。商户无需从空白画布开始搭建，可在默认样式基础上增删改，大幅降低使用门槛。默认样式变更仅影响新注册商户，不影响存量商户。

8. **未配置链接的卡片不可拖拽 + 免校验保存**：第三方活动卡片若跳转链接为空，在组件库中置灰且阻止拖拽，从源头杜绝无效卡片进入画布。因此保存时无需再做缺失路径校验，直接保存即可，简化保存流程。

### 待办/注意事项

- 组件缩略图需要开发在注册组件时一并提供（上传到 CDN）。缩略图设计规范待 UI 确认。
- 当运营修改了链接组件的活动后，已保存布局中的 `configSnapshot` 是否需要同步更新？建议采用"实时读取最新配置"方案，不存快照。
- C端权益页的实际渲染（带真实跳转）超出本文档范围，将在后续"小程序功能"和"H5 功能"文档中覆盖。
- 画布中的"双列布局"（colLeft / colRight）是否需要商户手动指定组件放到左列还是右列？当前设计为统一画布，自上而下排列。
