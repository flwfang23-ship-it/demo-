## 1. 需求背景与目标 (Why)

口令类推广内容无法在微信内直接传播，推广者需将口令嵌入H5落地页，用户打开H5后手动复制口令再到对应App使用。

不同活动使用不同H5样式是刚需，但运营不需要逐项配置颜色/字号/布局——开发预先做好若干套模板，运营只需给活动指定一个模板即可。系统自动生成H5链接，商户获取后即可投放。

## 2. 业务流程与逻辑 (How - 宏观)

### 2.1 总后台操作路径（🏢）

#### 路径A：管理H5模板

```
  进入H5模板列表      点击[新增模板]       填写模板信息        保存
  (B05-B列表)  ───►  (B05-B-a表单)  ───►  (填写表单)  ───►  (模板就绪)
       │                  │                  │                │
       │                  │                  │ 校验失败       │
       ▼                  ▼                  ▼                ▼
   (需运营登录)       (需开发提供模板编号)   标红提示        Toast"保存成功"
                                                           + 返回列表
```

#### 路径B：活动关联模板

```
  进入活动列表        找到已开启自动更新的活动   点击模板下拉    选择模板
  (B02列表)  ───►  (autoRefreshToken=true)  ───►  展开下拉  ───► 选中
       │                                             │              │
       │                                             │              │ 保存
       ▼                                             ▼              ▼
  (需运营登录)                                   展示已启用模板   H5链接自动生成
                                                 + 空选项"不关联"
```

| 步骤 | 角色 | 触发动作 | 所在页面 | 前置条件 | 后续页面 |
|------|------|----------|----------|----------|----------|
| 1 | 🏢 运营 | 管理模板 | B05-B 模板列表 | 已登录运营账号 | - |
| 2 | 🏢 运营 | 新增/编辑模板 | B05-B 列表 | 开发已提供模板编号 | B05-B-a 表单 |
| 3 | 🏢 运营 | 保存模板 | B05-B-a | 必填项通过校验 | 返回列表 |
| 4 | 🏢 运营 | 进入活动管理 | B02 活动列表 | 已登录 | - |
| 5 | 🏢 运营 | 为活动选择H5模板 | B02 列表/编辑表单 | 活动 autoRefreshToken=true | 下拉选择模板 |
| 6 | 🏢 运营 | 保存关联 | B02 表单 | 模板已选 | H5链接自动生成 |

### 2.2 客户端操作路径（🏪）

```
  进入推广链接抽屉    点击[生成推广链接]     查看推广结果
  (C01 Drawer)  ───►  (调取链接口)  ───►  (分组展示)
       │                                        │
       │                                        ├─ 小程序链接 / 唤起协议 / ...
       │                                        ├─ 团口令
       │                                        └─ 🆕 H5推广链接  ← 仅活动已关联模板时出现
       │                                             │
       │                                             │ 点击复制
       │                                             ▼
       │                                        Toast "已复制"
       ▼
  (需登录)
```

| 步骤 | 角色 | 触发动作 | 所在页面 | 前置条件 | 后续页面 |
|------|------|----------|----------|----------|----------|
| 1 | 🏪 商户 | 打开推广链接抽屉 | C01 工作台 | 活动已上架 | Drawer 滑出 |
| 2 | 🏪 商户 | 点击[生成推广链接] | Drawer | 已选推广位 | 返回各渠道链接 |
| 3 | 🏪 商户 | 复制H5推广链接 | Drawer | 活动已关联H5模板 | 剪贴板 + Toast |

### 2.3 C端用户访问路径（📱）

```
  用户点击H5链接      H5页面加载            查看口令             复制口令
  (微信内打开)  ───►  (读模板+token_cache)  ───► (模板渲染)  ───► (复制到剪贴板)
       │                  │                    │                  │
       │ H5未配置/下线    │ 模板不存在         │ 口令过期         │ 复制失败
       ▼                  ▼                    ▼                  ▼
   "活动已结束"        "页面不存在"      黄色过期提示         降级处理
                                            + 仍可复制         (长按手动复制)
```

  │   未关联      │─────────────────────►│   已关联      │

                                               │

**状态说明表**：

| 状态名称 | 状态含义 | 进入条件 | 可触达的状态 | 操作角色 |
|----------|----------|----------|-------------|---------|

## 6. 📱 H5展示页----------------------------------------------------------------------------------------------------------------------

> H5展示页是C端用户实际看到的落地页。页面由后端根据关联的模板代码渲染，模板文件由开发维护。

### 6.1 页面：口令H5展示页（页面ID: H5-P01）

##### 6.1.1 前置条件

| 条件项 | 说明 |
|--------|------|
| 访问方式 | 通过H5链接在微信内置浏览器或手机浏览器打开 |
| 活动状态 | 活动未下架 + 已关联H5模板 + 模板已启用 |
| 口令数据 | token_cache 中有对应活动的缓存数据 |

##### 6.1.2 页面加载流程

```
  用户打开 H5链接
  https://h5.example.com/{activityId}
       │
       ▼
  后端 API-H5P-01 处理：
  ① 根据 activityId 查关联的 h5TemplateId
  ② 查 tokenCache 获取口令数据
  ③ 将口令数据注入模板变量
  ④ 返回渲染后的 HTML
       │
       ▼
  浏览器展示完整H5页面
  (模板样式 + 新鲜口令 + 引导文案)
```

##### 6.1.3 模板变量规范

开发制作模板时，使用以下占位变量，后端渲染时替换：

| 变量名 | 来源 | 说明 |
|--------|------|------|
| `{{pageTitle}}` | 活动表 activityName | H5页面标题 |
| `{{supplier}}` | 活动表 supplier | meituan / eleme |
| `{{supplierName}}` | 映射 | 美团 / 饿了么 |
| `{{groupToken}}` | tokenCache.tokenData | 美团团口令（仅美团） |
| `{{tbWatchword}}` | tokenCache.tokenData | 淘宝口令-短（仅饿了么） |
| `{{tbWatchwordSuggest}}` | tokenCache.tokenData | 淘宝口令-建议版（仅饿了么） |
| `{{tokenUpdatedAt}}` | tokenCache.updatedAt | 口令更新时间 |
| `{{isStale}}` | 计算 | 是否超过15天 |
| `{{appName}}` | 映射 | supplier=meituan→"美团App"；eleme→"淘宝App" |

##### 6.1.4 页面组件状态

| 组件编号 | 状态 | 视觉表现 | 触发条件 |
|----------|------|----------|----------|
| C-H5P-01 | 正常 | 完整展示模板内容 | 模板存在 + token_cache 命中 + 未过期 |
| C-H5P-02 | 口令过期提示 | 顶部黄色提示条"口令可能已过期，请尝试复制后打开App验证" | is_stale=true |

| 规则编号 | 规则描述 | 适用层级 | 触发条件 | 处理逻辑 | 异常处理 |

## 8. 开发侧：H5模板制作规范

> 本节供前端开发参考。

### 8.1 模板文件

模板文件存放于 `h5-templates/` 目录，文件名为 `{code}.html`（与数据库 code 字段一致）。

```
h5-templates/
├── TPL_MEITUAN_001.html    ← 美团通用版
├── TPL_ELEME_001.html      ← 饿了么红包版
├── TPL_COMMON_001.html     ← 通用简洁版
└── ...
```

### 8.2 模板语法

模板使用简单的 `{{变量名}}` 占位语法，后端渲染时做字符串替换。

必含变量：
- `{{page_title}}` — 页面标题
- `{{group_token}}` — 美团团口令（美团模板必用）
- `{{tb_watchword}}` — 淘宝口令短版（饿了么模板必用）
- `{{tb_watchword_suggest}}` — 淘宝口令建议版（饿了么模板必用）
- `{{app_name}}` — 目标App名称（美团App / 淘宝App）

### 8.3 模板规范

| 规范项 | 要求 |
|--------|------|
| 视口 | `<meta name="viewport" content="width=device-width, initial-scale=1.0">` |
| 基准宽度 | 375px（移动端适配） |
| 依赖 | 模板自包含 CSS/JS，不依赖外部框架 |
| 复制功能 | 模板内必须包含口令复制按钮，JS 处理 clipboard 写入 + 降级方案 |
| 过期提示 | 需判断 `{{is_stale}}`，为 true 时展示过期提示条 |
| 空值兜底 | 口令变量为空时，对应区域展示"口令准备中" |
| 微信适配 | 需处理微信内置浏览器的 clipboard 兼容（降级为长按选中手动复制） |

## 模块汇总

> 1. **H5模板由开发预置，运营只选不配**：模板样式代码由开发维护，运营在后台模板管理页面录入模板名称、编号、预览图，然后在活动列表中下拉选择关联。
> 2. **关联即生效**：运营在活动列表下拉选择一个模板，系统即时生成H5链接。切换模板即时切换链接对应的样式。清空选择即时失效。
> 3. **最小化改动范围**：前端活动列表加一个下拉列，后端活动表加一个字段。不改动活动编辑表单的复杂结构。
> 4. **商户端无感获取**：商户在推广链接抽屉点[生成推广链接]，如果活动已关联H5模板则多一行"H5推广链接"，没关联则不出现。不需要商户做任何额外操作。
> 5. **模板停用不等同删除**：停用的模板新活动不可选，已关联的活动不受影响。
