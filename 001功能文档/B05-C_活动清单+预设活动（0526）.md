## 总后台预设活动配置（🏢）

---

### 1. 需求背景与目标 (Why)

运营团队需要为全平台商户预设一批"推荐活动"——当商户创建新的推广位时，这些活动的取链信息自动写入该推广位的取链清单，商户无需逐个手动添加。

这样做的价值：
- **降低商户操作门槛**：新建推广位后取链清单已有内容，商户"开箱即用"
- **运营引导**：运营可以将重点推广的活动预设进去，引导商户优先推广
- **减少遗漏**：商户不会因为不知道某个活动而漏掉推广机会

---

### 2. 业务流程与逻辑 (How - 宏观)

#### 2.1 总后台操作路径（🏢）

```
  进入活动管理        找到目标活动        打开预设开关
  (总后台→活动管理) ──► 列表行 ──► 标记为预设活动
       │                    │
       │                    │ 关闭预设开关
       │                    ▼
       │               取消预设标记
       │
       ▼
  (需管理员登录)      实时生效，影响后续
                     所有商户新建推广位
```

| 步骤 | 角色 | 触发动作 | 所在页面 | 前置条件 | 后续影响 |
|------|------|----------|----------|----------|----------|
| 1 | 🏢 运营 | 进入活动管理列表 | 总后台活动管理页 | 管理员登录 | — |
| 2 | 🏢 运营 | 打开活动的预设开关 | 列表行内操作 | 活动已上架 | 后续所有商户创建推广位时，该活动自动入列 |
| 3 | 🏢 运营 | 关闭活动的预设开关 | 列表行内操作 | — | 后续新建推广位不再自动入列；已入列的链接不受影响 |

---

#### 2.2 状态流转

```
  ┌──────────────┐   运营打开开关        ┌──────────────┐
  │  非预设       │────────────────────►│  预设中        │
  │  isPreset    │                      │  isPreset    │
  │  = false     │◄────────────────────│  = true      │
  │              │   运营关闭开关        │              │
  └──────────────┘                      └──────┬───────┘
                                               │
                                               │ 商户创建推广位
                                               ▼
                                        ┌──────────────┐
                                        │ 静默写入       │
                                        │ 取链清单       │
                                        │ (商户无感知)   │
                                        └──────────────┘
```

| 状态 | 含义 | 进入条件 | 影响范围 |
|------|------|----------|----------|
| 非预设 | 普通活动，不触发自动入列 | 默认状态 / 运营关闭开关 | — |
| 预设中 | 平台预设活动，商户创建推广位时自动入列 | 运营打开开关 | 全平台所有商户 |

## 创建推广位静默入列（🔗 系统自动）

---

### 1. 需求背景与目标 (Why)

商户创建推广位后，如果取链清单为空，商户需要逐个手动添加活动链接——操作成本高，且可能遗漏重点活动。

系统在推广位创建时**静默**将总后台预设的活动写入取链清单。商户打开取链清单时链接已在其中，直接可用。整个过程商户无感知。

---

### 2. 业务流程与逻辑 (How - 宏观)

#### 2.1 触发时机

| 触发场景 | 触发位置 | 说明 |
|----------|----------|------|
| 商户手动创建推广位 | 推广位管理 → 新增推广位弹窗 → [创建] | 推广位写入成功后，立即触发静默入列 |
| 商户首次打开取链清单 | 推广位列表 → [查看(N)] | 如果取链清单为空，兜底填充预设活动（覆盖首次打开时数据尚未入列的边界情况） |

> **不触发**：系统注册时自动创建的默认推广位（如 PROMO_001）**不触发**静默入列——那是系统预置推广位，不应自动绑活动。

#### 2.2 执行流程

```
  商户点击[创建]        后端写入推广位        查询预设活动          逐条调取链接口        写入取链清单
  ──────────────────►  成功返回            isPreset = true       美团/饿了么API       promoChainList
       │                    │                    │                    │                    │
       │                    │                    │                    │                    │
       ▼                    ▼                    ▼                    ▼                    ▼
  ┌──────────┐        ┌──────────┐        ┌──────────┐        ┌──────────┐        ┌──────────┐
  │ 前端校验  │        │ 写入DB   │        │ SELECT   │        │ 传入     │        │ INSERT   │
  │ 名称+分组 │        │ 返回ID   │        │ FROM     │        │ platform │        │ INTO     │
  └──────────┘        └──────────┘        │ activity │        │ Config   │        │ chain    │
                                           │ WHERE    │        │ +actId   │        │ list     │
                                           │ isPreset │        │ → 真实链接│        │ (去重)   │
                                           │ = true   │        └──────────┘        └──────────┘
                                           └──────────┘
```

> **关键变更**：预设活动入列时，不是用模板字段拼链接，而是**逐条调美团/饿了么取链接口**获取真实链接数据，与 B05 手动取链走同一套接口、同一套字段映射规则。

#### 2.3 入列数据来源

每条预设活动入列时，后端调美团/饿了么取链接口获取真实链接数据，写入取链清单。

**数据来源**：美团/饿了么取链接口（与 B05 手动取链接口一致），传入活动对应的 `platformConfig` + `activityId`，获取真实返回数据。

**字段映射**（美团/饿了么返回 → 取链清单扁平字段），详见 B05 文档 3.4 节：

**美团**：

| 取链清单字段 | 美团来源 |
|-------------|---------|
| `shortUrl` | key `1`（短链接） |
| `longUrl` | key `2`（长链接） |
| `deeplink` | key `3`（唤起协议） |
| `miniPath` | key `4`（小程序路径） |
| `groupToken` | key `5`（团口令） |
| `qrcode` | key `6`（小程序二维码） |

**饿了么**：

| 取链清单字段 | 饿了么来源 |
|-------------|-----------|
| `longUrl` | `taobao_promotion.h5_url` |
| `deeplink` | `taobao_promotion.scheme_url` |
| `groupToken` | `taobao_promotion.tb_watchword` |
| `searchToken` | `taobao_promotion.tb_watchword_suggest` |
| `qrcode` | `taobao_promotion.tb_qr_code` |
| `promoImage` | `taobao_promotion.img_url` |
| `fullGroupToken` | `elemedata.link.taobao_word` 或 `full_taobao_word` |
| `miniQrcode` | 顶层 `tb_mini_qrcode` |
| `tbSchemeUrl` | 顶层 `tb_scheme_url` |

> **与 B05 手动取链一致**：静默入列和 B05 手动取链走同一套取链接口、同一套字段映射。取链清单中所有链接字段的数据来源统一为美团/饿了么接口返回的真实数据。

#### 2.4 去重规则

入列前检查是否已存在同 `promoId + activityId` 的记录。已存在则跳过。

#### 2.5 商户感知

**零感知**。商户只在打开取链清单时看到已有链接——不会被告知这些链接是"预设活动"自动生成的。对商户而言，这些就是"推广位自带的活动链接"。

## 取链清单入口（🏪 推广位 → 查看/管理取链）

---

### 1. 需求背景与目标 (Why)

商户需要对每个推广位管理其下的活动取链链接。取链清单入口提供了"推广位 → 链接管理"的桥梁——商户在推广位列表中点击按钮即可进入该推广位的全屏链接管理视图。

链接来源有两种（对商户透明）：
- 系统静默入列的（来自预设活动）
- 商户手动新增的

两种来源在清单中地位平等，均可查看、复制、删除。

## 模块汇总

> 1. **预设活动是平台级运营配置**：总后台运营通过活动列表中的开关标记/取消预设活动。预设标记全平台生效，对所有商户透明。
> 2. **商户端完全无感**：活动清单不显示预设标识、创建推广位不弹提示、取链清单不区分链接来源。商户只知道"取链清单里有链接可用"。
> 3. **静默入列调取链接口获取真实数据**：创建推广位时后端查询预设活动 → 逐条调美团/饿了么取链接口 → 将真实返回数据写入取链清单。与 B05 手动取链走同一套接口、同一套字段映射。
> 4. **B05 取链结果回写取链清单**：商户在 B05 抽屉中手动取链成功后，真实链接数据同步回写覆盖取链清单对应记录。取链清单始终保存最新一次取链的真实结果。
> 5. **取消预设不回删**：运营关闭预设只影响后续新建推广位，已入列的链接不受影响。
> 6. **取链清单是商户的链接管理中心**：支持按平台过滤、搜索、展开详情、复制、新增、删除。链接来源对商户不可见。
