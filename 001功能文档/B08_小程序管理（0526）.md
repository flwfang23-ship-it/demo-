## 功能 B08：小程序管理

---

### 1. 需求背景与目标 (Why)

商户通过 SaaS 平台开展微信小程序业务，需要将小程序 appid 纳入平台统一管理。平台提供两条入驻通道——「快速注册」（直接创建已认证的正式小程序）和「试用小程序」（14 天免费体验后再转正），无论走哪条通道，产出的 appid 最终都落入「小程序管理」模块进行全生命周期管理。

核心目标：
- **双通道统一纳管**：快速注册和试用小程序两条通道产出的 appid 统一列表展示，通过来源标签区分
- **全生命周期覆盖**：从入驻 → 代码上传 → 提审核 → 发布 → 配置变更 → 解绑，全流程在 SaaS 内闭环
- **双层权限隔离**：总后台看全平台所有小程序（可介入管理），客户端后台只看自己的小程序

#### 2.3 小程序来源状态流转图

```
商户入驻小程序
      │
      ├── 快速注册通道（B08-B）
      │     │
      │     ├── DRAFT（草稿，未提交）
      │     │     ↓ 提交微信 create
      │     ├── SUBMITTED（已提交，等待法人操作）
      │     │     ↓ 法人核验中
      │     ├── VERIFYING（核验中）
      │     │     ↓ 收到 appid
      │     ├── SUCCESS → 进入小程序管理（正式小程序）
      │     │
      │     └── FAIL（失败/驳回/超时）
      │
      └── 试用通道（B08-A）
            │
            ├── CREATING（生成授权页中）
            │     ↓ 24h 内管理员扫码
            ├── WAIT_ADMIN_AUTH（等待管理员确认）
            │     ↓ 收到 appid
            ├── TRIAL_READY（体验期，14天）
            │     │
            │     ├── 到期前3天 → TRIAL_EXPIRING_SOON（即将到期）
            │     │
            │     ├── 发起转正 → UPGRADING
            │     │     ↓ 法人扫脸+管理员确认
            │     │     ├── UPGRADE_FAILED → 可重新发起
            │     │     └── OFFICIAL（转正完成，变为正式小程序）
            │     │
            │     └── 到期未转正 → TRIAL_EXPIRED（已注销）
            │
            └── 正式小程序管理（统一管理）
                  │
                  ├── 代码管理：上传 → 提审 → 审核中 → 审核通过/驳回 → 发布
                  ├── 成员管理：绑定体验者/开发者
                  ├── 域名配置：业务域名/request域名等
                  └── 解绑：解除小程序与商户的绑定关系
```

**小程序管理状态说明表**：

| 状态名称 | 状态含义 | 进入条件 | 可触达的状态 | 操作角色 |
|----------|----------|----------|-------------|---------|
| DRAFT | 快速注册草稿 | 商户开始填写注册信息 | SUBMITTED | 🏪 商户 |
| SUBMITTED | 已提交微信 | 商户提交注册表单成功 | VERIFYING, FAIL | 🏪 商户 |
| VERIFYING | 法人核验中 | 微信通知法人已进入核验 | SUCCESS, FAIL | 微信侧 |
| SUCCESS | 注册成功（正式） | 收到微信推送 appid | 代码管理相关状态 | 系统自动 |
| FAIL | 注册失败 | 法人超时/信息不一致/驳回 | DRAFT（重新发起） | 🏪 商户 |
| CREATING | 试用创建中 | 商户发起试用创建 | WAIT_ADMIN_AUTH | 🏪 商户 |
| WAIT_ADMIN_AUTH | 等待管理员确认 | 授权页生成成功 | TRIAL_READY, FAIL | 🏪 商户（管理员） |
| TRIAL_READY | 体验期 | 管理员确认，收到 appid | TRIAL_EXPIRING_SOON, UPGRADING, TRIAL_EXPIRED | 系统自动 |
| TRIAL_EXPIRING_SOON | 即将到期（≤3天） | 距到期日 ≤ 3 天 | UPGRADING, TRIAL_EXPIRED | 系统自动 |
| UPGRADING | 转正中 | 商户发起转正 | OFFICIAL, UPGRADE_FAILED | 🏪 商户 + 微信侧 |
| UPGRADE_FAILED | 转正失败 | 法人拒绝/超时 | UPGRADING（重新发起） | 🏪 商户 |
| OFFICIAL | 正式小程序 | 转正完成 / 快速注册成功 | 代码管理相关状态 | 系统自动 |
| TRIAL_EXPIRED | 试用已过期 | 14 天到期未转正 | - | 系统自动 |

## 模块汇总

**关键设计决策**：
1. 小程序管理作为统一入口模块，承载两条入驻通道（B08-A 试用、B08-B 快速注册）产出的小程序资产
2. 来源字段（sourceType）区分正式注册 vs 试用小程序，列表页按来源分组展示
3. 总后台拥有全平台视图，可介入管理任意商户的小程序；客户端后台严格按 tenantId 隔离
4. 试用期内功能降级（支付/直播/手机号），通过前端入口置灰 + 引导转正来处理
5. 转正完成后必须引导商户重设小程序名称（去后缀），否则后续提审会报错
