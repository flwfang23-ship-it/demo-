# 试用小程序通道：B端功能模块规格

> 本文档按 G 模板（B端）结构编写，覆盖商户通过 SaaS 平台创建 14 天试用小程序 → 体验 → 转正的完整流程。本功能产出的 appid 自动纳入 B08 小程序管理模块统一管理。

---

## 模块概览

| 项目 | 内容 |
|------|------|
| 模块名称 | 试用小程序通道（14天体验 → 转正） |
| 所属层级 | 🏪 客户端后台（主要操作层）+ 🏢 总后台（查看/介入管理） |

## 功能 B08-A：试用小程序通道

---

### 1. 需求背景与目标 (Why)

商户在决定长期购买 SaaS 服务前，希望能先"真实体验"小程序前台的效果（页面装修、商品展示、下单流程等），而不是仅看 Demo。平台通过微信第三方平台的「试用小程序」能力，让商户在 1 分钟内快速获得一个 14 天有效期的试用小程序 AppID，零门槛体验完整功能链路。体验满意后，通过「转正」流程（法人扫脸认证）将试用小程序变为正式长期可用的小程序。

核心目标：
- **极速体验**：填最少信息（名称 + 头像），管理员扫码确认，1 分钟内拿到可用小程序
- **真实试跑**：14 天内可真实体验页面装修、模板下发、预览版功能
- **无缝转正**：体验满意后一键发起转正，走法人认证流程，完成后自动变为正式小程序
- **功能降级透明**：试用期不支持的能力（支付/直播/手机号登录等）在界面明确告知并引导转正

### 2. 业务流程与逻辑 (How - 宏观)

#### 2.1 总后台操作路径（🏢）

> 总后台在此功能中主要是"查看"角色，平台运营人员可查看全平台试用小程序的列表、状态和转正进度。

```
总后台运营登录
      │
      ▼
进入小程序管理列表（筛选来源=试用小程序）
      │
      ├── 查看所有试用小程序的状态和到期时间
      │
      ├── 查看试用小程序的转正进度
      │
      └── 导出试用小程序报表（到期预警、转正转化率等）
```

| 步骤 | 角色 | 触发动作 | 所在页面 | 前置条件 | 后续页面 |
|------|------|----------|----------|----------|----------|
| 1 | 🏢 运营 | 进入小程序列表 | miniapp-list-admin | 已登录总后台 | - |
| 2 | 🏢 运营 | 筛选来源=试用小程序 | miniapp-list-admin | 列表已加载 | - |
| 3 | 🏢 运营 | 查看详情 | miniapp-detail | 列表有数据 | 详情页 |

```

            │     ▼

            │     │
            │     ▼

            │     │

            │
            ▼

            │
            ▼
      微信回调 → SaaS 收到新 AppID
            │
            ▼

            │
            ▼
      Step B：14 天体验期
            │
            ├── 在「我的小程序」列表中显示 [试用·剩余N天]
            │
            ├── 可操作：代码管理（上传/预览）、成员管理（添加体验者）、页面装修
            │
            ├── 不可用（试用限制）：微信支付、小程序直播、被搜索、手机号快捷登录
            │     └── 点击时弹出引导转正弹窗
            │
            ├── 到期前3天：列表行高亮 + 站内信提醒
            │
            └── 到期未转正：小程序被微信注销 → 列表标记灰色"已过期"
            │
            ▼
      Step C：转正（trial-upgrade）
            │
            ├── 点击「立即转正」按钮
            │     │
            │     ▼
            │   进入转正表单页
            │     │
            │     ├── 填写/核对：
            │     │   · 企业名称（必须与工商登记一致）
            │     │   · 统一社会信用代码（18位）
            │     │   · 法人姓名
            │     │   · 法人微信号
            │     │   · 联系人/电话
            │     │
            │     ├── 提交 → 调微信 verifyBetaWeapp 接口
            │     │
            │     ▼
            │   状态变为 UPGRADING（转正中）
            │     │
            │     ├── 提示：转发给法人微信完成人脸核验
            │     │
            │     ├── 法人完成核验 → 管理员收到确认通知
            │     │
            │     └── 管理员确认 → 转正完成
            │           │
            │           ▼
            │   状态变为 OFFICIAL（正式）
            │           │
            │           ├── [关键步骤] 引导商户重设小程序名称（去掉试用后缀）
            │           │     └── 不重设的话后续提审会报错
            │           │
            │           ├── 标签从 [试用] 切换为 [正式]
            │           │
            │           ├── 清空到期倒计时
            │           │
            │           ├── 解锁全部功能（支付/直播/手机号）
            │           │
            │           └── 引导补充正式配置（客服入口、类目、支付开通等）
            │
            └── 转正失败（法人拒绝/超时）→ 可重新发起
```

**状态说明表**：

| 状态名称 | 状态含义 | 进入条件 | 可触达的状态 | 操作角色 |
|----------|----------|----------|-------------|---------|
| CREATING | 生成授权页中 | 商户提交创建请求 | WAIT_ADMIN_AUTH, FAIL | 🏪 商户 |
| WAIT_ADMIN_AUTH | 等待管理员扫码确认 | 授权页 URL 生成成功 | TRIAL_READY, FAIL | 🏪 商户（管理员扫码） |
| TRIAL_READY | 体验期（14天有效） | 管理员确认，微信推送 appid | TRIAL_EXPIRING_SOON, UPGRADING, TRIAL_EXPIRED | 系统自动 |
| TRIAL_EXPIRING_SOON | 即将到期（≤3天） | trialExpireAt - now ≤ 3 | UPGRADING, TRIAL_EXPIRED | 系统自动 |
| UPGRADING | 转正流程中 | 商户发起转正，法人扫脸+管理员确认中 | OFFICIAL, UPGRADE_FAILED | 🏪 商户 + 微信侧 |
| OFFICIAL | 转正完成（正式小程序） | 法人核验+管理员确认均通过 | - | 系统自动 |
| UPGRADE_FAILED | 转正失败 | 法人拒绝/超时/信息不一致 | UPGRADING（重新发起） | 🏪 商户 |
| TRIAL_EXPIRED | 试用到期已注销 | 14天到期未转正，小程序被微信注销 | - | 系统自动 |
| FAIL | 授权超时/失败 | 24h 内未扫码确认或其他错误 | CREATING（重新生成） | 🏪 商户 |

#### 5.3 业务规则

| 规则编号 | 规则描述 | 适用层级 | 触发条件 | 处理逻辑 | 异常处理 |
|----------|----------|----------|----------|----------|----------|
| RULE-TR-001 | 试用配额每个微信号最多 5 个 | 🏪 客户端 | 创建试用小程序 | 创建前查配额 → 达上限禁止创建 | 提示限额 + 引导解绑 |
| RULE-TR-002 | 授权页 24h 有效 | 🏪 客户端 | 授权页生成后 | 24h 倒计时 → 过期后允许重新生成 | 过期提示 + [重新生成] 按钮 |
| RULE-TR-003 | 授权页不能重复生成（进行中时） | 🏪 客户端 | 有进行中的授权任务 | 已有有效授权页时不允许多次生成 | 提示"已有进行中的授权，请先完成或等待过期" |
| RULE-TR-004 | 试用期 14 天 | 两层皆有 | 收到 appid 时 | 按 appid 获取时间 + 14 天计算 trialExpireAt | - |
| RULE-TR-005 | 试用期功能降级 | 🏪 客户端 | sourceType=trial | 支付/直播/手机号入口置灰或弹窗引导转正 | 引导转正 |
| RULE-TR-006 | 到期前 3 天提醒 | 🏪 客户端 | trialExpireAt - now ≤ 3 天 | 列表行高亮 + 站内信 + 短信通知 | - |
| RULE-TR-007 | 转正后必须重设名称 | 🏪 客户端 | 转正完成时 | 弹窗强制引导（不可跳过） → 调设置名称接口去掉试用后缀 | 提示"名称含试用后缀，请重新设置" |
| RULE-TR-008 | 转正失败可重试 | 🏪 客户端 | 状态为 UPGRADE_FAILED | 保留上次填写的信息 → 允许重新发起 | 注意频控，避免频繁调用微信接口 |
| RULE-TR-009 | 法人步骤不要卡死 | 🏪 客户端 | 转正进入 UPGRADING 后 | 提供转正进度查询 + 法人未操作时的重新通知入口 | - |
| RULE-TR-010 | 解绑限制 | 🏪 客户端 | 试用小程序的体验成员解绑 | 最多解绑 5 次 | 提示剩余解绑次数 |

### 6. 关键约束 & 踩坑清单（研发/测试必读）

| 约束项 | 说明 | 影响 |
|--------|------|------|
| 名称强制后缀 | 试用阶段名称自动带"的体验小程序"等后缀 | 转正后必须调用设置名称接口去除，否则提审报错 |
| 不支持微信支付 | 试用期无支付能力 | 支付相关页面做降级处理 |
| 不支持小程序直播 | 试用期无直播能力 | 直播入口置灰 |
| 不能被搜索 | 试用小程序不出现在微信搜索结果中 | 分发策略需调整 |
| 无 appsecret | 部分后端能力不可用 | 依赖 appsecret 的功能需做兼容 |
| 手机号接口不可用 | 无法调用微信手机号快捷登录 | 改为让用户手动输入手机号走 SaaS 客户库 |
| 无法生成小程序码 | 只能用普通二维码 | 普通二维码不能长按识别，入口改为搜索/消息卡片/H5中转 |
| 不可扫码登录 mp 后台 | 试用版不走传统扫码绑定管理员姿势 | 管理操作全部走 SaaS 后台，不依赖 mp.weixin.qq.com |
| 体验成员上限 15 人 | 最多添加 15 个体验者 | 提示限额 |
| 解绑次数上限 5 次 | 体验成员解绑总次数 ≤ 5 | 提示剩余次数 |
| 转正后可免邮箱密码登录 | 转正后可用扫码方式登录 mp 后台 | 便利性卖点 |
| 法人核验通知方式变更 | 平台下发给服务商 URL 而非模板消息 | SaaS 需自己把 URL 做成二维码/页面给商户 |

## 模块汇总

**关键设计决策**：
1. 试用小程序作为"零门槛体验入口"，让商户在 1 分钟内拿到可用小程序，降低决策门槛
2. 授权页改为二维码展示 + 24h 倒计时，适配微信不再发模板消息的变化
3. 试用期功能降级采用"入口保留 + 引导转正"策略（而非直接隐藏），提升转正转化率
4. 转正后强制引导重设名称，作为转正完成流的必做步骤，避免后续提审报错
5. 到期前 3 天 + 1 天双重提醒，SaaS 侧兜底（不依赖微信通知）
