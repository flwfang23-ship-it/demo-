## C01 首页推广工作台 - 实现清单

### 模块② 昨日数据（收入概览区）
- 昨日预估佣金(元) + 昨日订单数，32px Bold 大字展示
- 环比变化（↑/↓ + 百分比），正绿色/负红色
- 无数据时显示"--"，无环比时不展示
- 点击卡片跳转日报列表页（reports-list）
- 数据来源：`MockData.latestReport`

### 模块③ 财务数据（资金提现区）
- 渐变蓝色卡片（#2563EB→#1D4ED8），白色文字
- 账户余额 36px Bold 展示
- [提现]按钮（余额≤0置灰）
- 底部：提现审核中 / 已提现 副指标
- 未绑定账户状态：显示"请先绑定提现账户" + [去绑定]
- 提现弹窗：输入金额 → 前端校验（≥100, ≤余额）→ 模拟提交 → Toast反馈
- 数据来源：`MockData.financeData`

### 模块① 取链模块（业务推广核心区）
- 供应商Tab栏：全部 / 美团联盟 / 饿了么联盟（动态生成）
- Tab切换为前端本地过滤，不重新请求
- 活动卡片网格：3列自适应（≤1400px缩为2列）
- 卡片内容：主图占位(16:9) / 主标题(2行截断) / 副标题 / 供应商Tag(美团红/饿了么蓝) / 佣金高亮 / 有效期 / [立即推广]
- 空状态：全部空 + 单个Tab空 两种提示
- 取链弹窗：Step1选择推广位+链接类型 → Step2展示推广链接+复制/关闭
- 数据来源：`MockData.availableActivities`

## C02 小程序管理 - 实现清单

### 列表页（C02-A）
- 表格列：复选框 / 小程序名称 / AppID / 小程序类型 / 部署状态 / 订阅开关 / 注册时间 / 状态 / 操作
- 默认排序：注册时间倒序
- 分页：5条/页（演示用），含上一页/下一页/页码
- 搜索：小程序名称/AppID模糊匹配
- 筛选：状态筛选（全部/正常/审核中/已停用）+ 订阅状态筛选（全部/已开启/已关闭）
- 部署状态：已上线(绿) / 审核中(橙) / 部署中(蓝)
- 状态标签：正常(绿) / 审核中(黄) / 已停用(灰)

### 订阅开关（C02-C）
- 列表行内 Switch 开关，点击即时切换
- 批量操作：勾选多行 → 显示[批量开启]/[批量关闭]按钮
- 全选/取消全选支持
- 切换后 Toast 反馈

### 注册/编辑小程序（C02-A-a）
- 弹窗表单：小程序名称(必填≤30字) / AppID(必填,唯一校验) / 小程序类型(下拉必填) / 描述(选填≤200字)
- 编辑模式预填现有数据
- AppID 重复校验提示
- 新注册自动生成ID、默认开启订阅、部署状态"部署中"

### 小程序详情（C02-B）
- 基础信息网格展示（名称/AppID/类型/状态/部署/订阅/注册时间/描述）
- 部署进度时间线（提交授权→名称审核→备案审核→部署上线，含状态标记）
- 订阅开关操作记录摘要（最近5条）
- [查看全部]跳转完整操作记录弹窗

### 订阅操作记录（C02-C-a）
- 只读表格：操作时间 / 操作类型（标签区分开启/关闭） / 详情
- 数据来源：`MockData.operationLogs` 中订阅相关记录

## 接口约定（模拟）
- C01三大模块数据通过 `setTimeout` 模拟并行加载（100ms/150ms/200ms），单个失败不影响其他模块
- C02列表数据同步读取 MockData，搜索筛选分页为前端本地计算
- 提现/取链/注册等写操作均为前端模拟，含 Toast 反馈
