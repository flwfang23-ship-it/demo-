## 项目背景

小程序SaaS营销平台 — 客户管理后台 H5 Demo。纯 HTML+CSS+JS，无构建工具。PC 端管理后台，最小宽度 1200px，左侧侧边栏 + 右侧内容区布局。

**当前状态**：代码位于 `h5-demo/` 目录。

**模块清单**：

| 编号 | 模块 | 核心页面 |
|------|------|----------|
| C01 | 首页概览（商户推广工作台） | 昨日数据 + 财务数据 + 取链模块 |
| C02 | 小程序管理 | 列表 + 注册/编辑 + 详情 + 订阅开关 |
| C03 | 推广位管理 | 列表 + 新增/编辑 + 设为默认 + 删除 |
| C04 | 权益页配置（核心） | 左右分栏 + Tab + 6坑位网格 + 活动选择器弹窗 + 推广位管理弹窗 |
| C05 | 日报 | 列表 + 详情 |
| C06 | 账号设置 | 企业信息 + 修改密码 + 操作日志 |

## 需求2：推广分组管理（独立页签）

### 需求描述

在侧边栏「推广管理」菜单下新增 **"推广分组"** 页签，作为独立页面，提供推广分组的完整管理能力：

- **查询**：支持按分组名称关键词搜索
- **新增**：创建新的推广分组（名称不超过10个字符）
- **修改**：支持对已有分组进行重命名
- **删除**：删除指定分组（需二次确认，分组下有推广位时提示用户）

**约束**：推广位管理中已有的分组相关功能（分组Tab切换、新增/删除分组）**保留不动**，本需求是新增独立入口，两者共享同一套 `promotionGroups[]` 数据源。

### 涉及文件

| 文件 | 改动内容 |
|------|----------|
| [js/nav.js](h5-demo/js/nav.js) | 在 g02「推广管理」下新增菜单项 `g02-d` → `promotion-group-list` |
| `h5-demo/pages/promotion-group-list.html` | **新建** — 分组管理页面（搜索栏 + 分组列表表格 + CRUD弹窗） |
| [js/mock-data.js](h5-demo/js/mock-data.js) | 无需改动，复用 `promotionGroups[]` |
| [index.html](h5-demo/index.html) | 新增页面级样式（`.promo-group-page-*`）和初始化逻辑 |
| [h5-demo/CODE_MAP.md](h5-demo/CODE_MAP.md) | 新增「推广分组管理」条目 |

### 页面布局设计

```
┌─────────────────────────────────────────────────┐
│  搜索栏：[搜索分组名称...]           [+ 新增分组] │
├─────────────────────────────────────────────────┤
│  分组名称        │ 包含推广位数  │ 创建时间  │ 操作│
│  ├ 默认分组      │     12       │ 2025-01-06│ ✏️ 🗑│
│  ├ 618大促       │      5       │ 2025-05-20│ ✏️ 🗑│
│  └ 新客专享      │      0       │ 2025-06-01│ ✏️ 🗑│
├─────────────────────────────────────────────────┤
│  分页                                            │
└─────────────────────────────────────────────────┘
```

### 实现要点

1. **菜单注册**：`nav.js` 中 `g02` 的 children 追加 `{ id: 'g02-d', label: '推广分组', page: 'promotion-group-list' }`
2. **页面HTML**：新建 `promotion-group-list.html`，表格列：分组名称 / 包含推广位数 / 创建时间 / 操作（编辑、删除）
3. **搜索**：前端实时过滤，按分组名称模糊匹配
4. **新增弹窗**：输入分组名称 → 校验（非空、不重名、≤10字符） → 写入 `promotionGroups[]`
5. **编辑弹窗**：复用新增弹窗结构，预填当前名称，保存时更新
6. **删除**：`showConfirm` 二次确认，统计该分组下推广位数量并提示（如"该分组下有 5 个推广位，删除后它们将移至默认分组"）
7. **数据同步**：所有操作直接修改 `window.MockData.promotionGroups`，推广位管理页切换回来时通过重新渲染自动同步

### 风险 / 注意事项

- 删除分组时，其下的推广位需**迁移到默认分组**，不能直接删除
- 编辑/删除操作需同步更新 `promotionGroups[]` 中每个 promotion 的 `groupId`/`groupName` 引用
- 与推广位管理页共享数据源，需确保数据一致性（不引入两份拷贝）
- 页面样式需复用现有 design token（`variables.css` / `common.css`）
- **推广位管理中的分组Tab栏、新增分组、删除分组功能保持不动**

## 需求4：推广位路径字段迁移（权益页配置 → 推广位管理列表）（已完成）

### 需求描述

历史旧版本中，权益页配置（C04）右侧面板的推广位列表包含以下字段和操作，现需**全部迁移**到推广位管理（C03）的列表中。

**迁移的3个字段：**
- **小程序推广路径**（miniPath）：推广位对应的小程序页面路径
- **H5推广路径**（h5Path）：推广位对应的H5落地页URL
- **缺失H5**：统计该推广位下第三方活动中未配置小程序路径的活动数量（如 "✅ 全部已配置" / "⚠ 3个活动未配置"）

**迁移的编辑操作：**
- **缺失编辑**：点击可打开详情弹窗，逐行查看/编辑各第三方活动的小程序推广路径（行内编辑：默认只读，点击编辑按钮切换为输入框，支持单行保存）

**源头（权益页配置 → 需移除）：**
- 删除右侧面板 `initRightPanel()` / `renderPromoTable()` 整套推广位列表UI
- 删除推广位选择器弹窗 `openPromoPicker()` / `confirmPromoPicker()`
- 删除推广位详情弹窗 `openPromoDetail()` 及行内编辑逻辑 `toggleDetailRowEdit()` / `saveSingleDetailPath()` / `savePromoDetailPaths()`
- 删除关闭确认弹窗 `openCloseConfirm()` / `confirmClosePromo()`
- 删除内嵌新建推广位 `inlineCreatePromo()` / `buildInlineGroupSelect()`
- 删除小程序APPID清单弹窗 `openMpAppIdList()`
- 删除相关变量：`benefitPagePromos`、`pickerCheckState`、`closeConfirmPromoId`、`currentDetailPromoId`
- 移除 benefit-config.html 中相关HTML弹窗节点

**目标（推广位管理列表 → 新增）：**
- 列表表格新增3列：小程序推广路径、H5推广路径、缺失H5
- 操作列新增「编辑路径」按钮，点击弹出第三方活动路径编辑弹窗

### 实现要点

1. **推广位管理列表新增3列**：在 `promotion.js` 的 `renderPromoList()` 中，表格追加小程序路径、H5路径、缺失H5三列；路径过长时截断显示并用 `title` 属性悬浮展示完整内容，未配置显示灰色"未配置"
2. **缺失H5计算逻辑**：新增函数 `getMissingActivities(promoId)`，遍历 `thirdPartyActivities[]`，统计 `promoPaths[promoId]` 为空或未配置的第三方活动数量；全部已配显示绿色 `✅ 全部已配置`，有缺失显示橙色 `⚠ N个活动未配置`
3. **编辑路径弹窗**：从历史旧版 `benefit-config.js` 迁移 `openPromoDetail()` 逻辑到 `promotion.js`，展示推广位基本信息（名称/分组/小程序路径/H5路径）+ 第三方活动路径配置列表
4. **行内编辑交互**：每行默认只读展示路径文本 + 状态标签；点击「✏️ 编辑」按钮进入编辑态（文本替换为 `<input>`，按钮变为「💾 保存」）；点击保存后更新 `thirdPartyActivities[].promoPaths`，刷新弹窗列表和推广位列表的缺失H5列
5. **数据源**：推广位级别的 `miniPath`/`h5Path` 已在 `mock-data.js` 的 `promotionGroups[].promotions[]` 中存在（当前部分有值），无需改动数据结构
6. **权益页配置精简**：移除约360行代码，`slotPathsBox`（slot级别路径展示+复制按钮）**保留不动**——它展示的是 slot 维度的路径，与推广位维度的路径是不同概念
7. **benefit-config.html 清理**：移除右侧面板区域HTML、推广位选择器弹窗（`promoPickerModalOverlay`）、推广位详情弹窗（`promoDetailModalOverlay`）、关闭确认弹窗（`closeConfirmModalOverlay`）、小程序APPID清单弹窗（`mpAppIdListModalOverlay`）

### 风险 / 注意事项

- `benefit-config.js` 中 `refreshAllCardMissingModules()` 依赖 `benefitPagePromos` 和 `getMissingActivities()`，移除时需检查调用链一并清理
- 推广位管理页操作列按钮增多（编辑 + 删除 + 编辑路径），需保持 `white-space:nowrap` 防止换行，小屏时可考虑使用下拉菜单收纳
- 缺失H5计算遍历第三方活动，当前数据量（9个推广位 × 6个第三方活动）无性能问题，后续数据增长可加缓存
- 编辑路径弹窗中的修改需同步刷新推广位列表的缺失H5列和弹窗内状态标签，确保数据一致
- 迁移过程中注意 `promotion.js` 新增函数命名使用 `promoPath*` 前缀，避免与现有 `promo*` 函数混淆
- 旧版 `benefit-config.js` 中 `getMissingActivities()` 和 `getPromoById()` / `getAllPromos()` 迁移到 `promotion.js` 后需调整为该文件的私有函数
