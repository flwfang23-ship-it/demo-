## 一、字段英文命名规则

### 1.1 基本格式

所有业务字段统一使用 **camelCase**（小驼峰）。已废弃 snake_case，存量文档中的 snake_case 字段在数据字典中已全部迁移。

```
正确: merchantId, activityName, createdAt
错误: merchant_id, activity_name, created_at
```

### 1.2 按语义分类的命名模式

| 语义类别 | 命名模式 | 示例 | 说明 |
|---------|---------|------|------|
| ID 字段 | `xxxId` | `merchantId`, `slotId`, `activityId`, `groupId`, `recordId`, `appId` | 用于实体标识符，不带 `_` 前缀 |
| 布尔字段 | `isXxx` / `hasXxx` / `canXxx` | `isDefault`, `isPreset`, `hasPathConfig`, `canAutoRefresh` | `is`=状态标记, `has`=是否拥有, `can`=是否允许 |
| 时间字段（时间戳） | `xxxAt` | `createdAt`, `updatedAt`, `expireAt`, `refreshedAt`, `submittedAt`, `taggedAt` | 存储精确时间点 |
| 时间字段（展示） | `xxxTime` | `lastLoginTime`, `clickTime`, `orderTime` | 前端展示用格式化时间 |
| 枚举字段-状态 | `xxxStatus` | `accountStatus`, `authStatus` | 实体生命周期状态 |
| 枚举字段-类型 | `xxxType` | `activityType`, `sourceType` | 实体分类/来源类型 |
| 数量字段 | `xxxCount` / `xxxNum` | `promotionCount`, `orderCount`, `merchantCount`, `attributedUserCount` | `Count`优先 |
| 金额字段（分） | `xxxAmount` | `orderAmount` | 后端存储，整数 |
| 金额字段（元） | `xxxPrice` | — | 前端展示，可带小数 |
| 排序字段 | `sortOrder` | `sortOrder` | 全文统一，不用 `sort` 或 `order` |
| 名称字段 | `xxxName` | `activityName`, `companyName`, `slotName`, `contactName`, `subjectName` | 展示用名称 |
| 路径字段 | `xxxPath` | `h5Path`, `miniPath`, `defaultPath` | URL或小程序页面路径 |

### 1.3 枚举值命名规范

| 场景 | 格式 | 示例 |
|------|------|------|
| 双值状态 | 英文单词，小写 | `online` / `offline`, `enabled` / `disabled`, `normal` / `disabled` |
| 多状态生命周期 | 大写 snake_case（与微信API保持一致） | `DRAFT`, `SUBMITTED`, `TRIAL_READY`, `WAIT_ADMIN_AUTH` |
| 来源/类型 | 英文单词，小写 | `official` / `trial`, `meituan` / `eleme`, `title` / `link` |
| 归因类型 | 驼峰英文 | `promotionLink`, `h5Direct`, `share`, `organic` |

**规则**：枚举值优先用英文单词全称，不用缩写。生命周期状态值（如 `TRIAL_READY`）因与微信API交互，保留大写 snake_case 惯例。

## 二、实体命名规范

### 2.1 实体英文名

| 规则 | 说明 |
|------|------|
| 格式 | PascalCase |
| 单词选择 | 英文全称，不用缩写 |
| 唯一性 | 全文唯一，新增实体前查 `001_数据字典.md` 第二节实体总表 |

```
正确: PromotionSlot, ThirdPartyActivity, AllianceActivity
正确: MiniProgram (不用 MP)
正确: AttributionRecord (不用 AttrRec)
```

### 2.2 实体缩写

缩写仅用于**特定场景**，不作为实体的主标识名：

| 缩写 | 全称 | 使用场景 |
|------|------|---------|
| MCH | Merchant | 日志/代码注释中的简写 |
| MP | MiniProgram | 日志/代码注释中的简写 |
| TPA | ThirdPartyActivity | 活动ID前缀 `TPA_XXX` |
| TPG | ThirdPartyGroup | 分组ID前缀 `TG_XXX` |
| CPT | Component | 代码注释 |
| FR | FastRegisterTask | 代码注释 |

**原则**：文档正文中必须使用全称，缩写仅在日志、代码注释、ID前缀等空间受限场景使用。

## 三、术语命名规范

### 3.1 中文术语统一

| 规则 | 示例 |
|------|------|
| 实体名用全称 | 推广位（不用"推广"）、联盟活动（不用"联盟"） |
| 动作名用动词+名词 | 取链、归因、打标、转正、入驻 |
| 角色名用名词 | 平台运营、商户管理员、法人、C端用户 |
| 状态名用形容词 | 已上架、已禁用、体验期、即将到期 |

### 3.2 废弃术语

以下术语已在 `002_术语表.md` 标记为废弃，所有文档中不得再出现：

| 废弃术语 | 标准替代 |
|---------|---------|
| 客户（指商户） | 商户 |
| promotion（指推广位） | PromotionSlot / 推广位 |
| 商家 | 商户 |
| promotion_rules (snake_case) | promotionRules |
| autoUpdateToken | autoRefreshToken |
| activity_id | activityId |
| createTime | createdAt |

完整废弃术语清单见 `002_术语表.md` 第十节。

## 五、禁止事项

| 编号 | 禁止项 | 原因 | 反例 |
|------|--------|------|------|
| P-001 | 禁止拼音字段名 | 不可维护，无国际通用性 | `shanghuId` → 应用 `merchantId` |
| P-002 | 禁止 snake_case | 统一 camelCase 规范 | `activity_id` → 应用 `activityId` |
| P-003 | 禁止同一概念多英文名 | 造成数据字典冗余和引用混乱 | `autoUpdateToken` / `autoRefreshToken` 并存 |
| P-004 | 禁止缩写不统一 | 读者无法判断含义 | `attrRevenue` vs `attributionRevenue` |
| P-005 | 禁止中英混合 | 命名风格混乱 | `活动Id` → 应用 `activityId` |
| P-006 | 禁止字段名过长超40字符 | 影响代码可读性 | 控制在30字符以内为佳 |
| P-007 | 禁止使用保留字 | 避免与语言/框架关键字冲突 | `class`, `type`, `enum`, `delete` |

## 七、模块编号规范

| 层级 | 编号格式 | 示例 | 说明 |
|------|---------|------|------|
| 主模块 | B + 2位数字 | `B01`, `B02`, `B08` | 独立功能模块 |
| 子模块 | B + 2位数字 + -大写字母 | `B03-A`, `B05-B`, `B08-A` | 附属功能拆分 |
| 功能文档 | `BXX_功能名.md` | `B01_用户登录注册.md` | 源文档命名 |

**新增子模块规则**：当主模块文档超过800行或包含2个以上可独立描述的子功能时，拆分为子模块文档。

## 九、源文档命名冲突裁决汇总

以下已在 Step 1 中裁决，收录于此作为判例：

| 编号 | 冲突 | 裁决 | 判例价值 |
|------|------|------|---------|
| CON-001 | B01 camelCase vs B02 snake_case | 统一 camelCase | 存量 snake_case 全量迁移 |
| CON-003 | 客户端 `id` vs 总后台 `slotId` | 统一为 `slotId` | 总后台命名优先级高于客户端 |
| CON-004 | 客户端 `name` vs 总后台 `slotName` | 统一为 `slotName` | 同上 |
| CON-008 | B04 `id` vs B02 `activity_id` | 统一为 `activityId` | 跨模块同概念字段统一命名 |
| CON-009 | `autoUpdateToken` vs `autoRefreshToken` | 统一为 `autoRefreshToken` | 以最准确语义为准 |
| CON-011 | `isPreset` 同名字段不同实体 | 实体前缀已区分，保留 | 同名字段加实体前缀可区分时允许 |
| CON-014 | `FastRegisterTask.companyName` vs `Merchant.companyName` | 不同实体，允许同名 | 不同实体同名不视为冲突 |

## 十、维护规则

1. **本文档是命名规范的唯一来源**，所有新增字段和术语必须以此为准
2. **修改命名规范需同步更新**：改本文档 → 更新数据字典受影响字段 → 更新术语表受影响术语 → 回改业务文档中的引用
3. **命名争议提交路径**：在 `cache/step3_issues.md` 记录争议项，含双方论据，等待人工裁决
4. **版本号**：本文档随数据字典版本号联动更新，当前版本 v1.0
