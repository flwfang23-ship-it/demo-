## 一、命名规范总则

**字段命名规则**（全文统一）：

| 规则 | 格式 | 示例 |
|------|------|------|
| 基本格式 | camelCase | `merchantId` |
| ID 字段 | `xxxId` | `slotId`, `activityId`, `groupId` |
| 布尔字段 | `isXxx` / `hasXxx` / `canXxx` | `isDefault`, `isPreset`, `hasPathConfig` |
| 时间字段 | `xxxAt` (时间戳) / `xxxTime` (展示文本) | `createdAt`, `expireAt`, `submittedAt` |
| 枚举字段 | `xxxStatus` / `xxxType` | `accountStatus`, `sourceType` |
| 数量字段 | `xxxCount` | `promotionCount`, `orderCount` |
| 金额字段 | `xxxAmount` (分) / `xxxPrice` (元) | `orderAmount` |
| 排序字段 | `sortOrder` | `sortOrder` |

**禁止事项**：
- 禁止拼音字段名
- 禁止 snake_case（B02/B05-A 旧命名全部迁移）
- 禁止同一概念多英文名
- 禁止缩写不统一

## 七、归因规则速查

### 归因优先级（RULE-B03B-010）

```
推广链接直达 (promotionLink) > 分享裂变 (share) > H5页面参数 (h5Direct) > 自然流量-默认推广位 (organic)
```

### 小程序端 vs H5端差异

| 维度 | 小程序端 | 直推H5端 |
|------|---------|---------|
| 归因依据 | 用户标签（持久化） | 页面参数（即时） |
| 跨会话保持 | 是 | 否 |
| 分享传递 | 是（属性继承） | 否（按URL参数独立归因） |
| 标签覆盖 | 新链接覆盖旧标签 | 每次访问独立判断 |
| 推广位删除 | 回退到默认推广位 | 页面参数失效，归因默认 |
