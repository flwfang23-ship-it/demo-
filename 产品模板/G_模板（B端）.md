## B端后台页面通用规范

> 所有 B 端后台页面（总后台 + 客户端后台）应遵循以下通用交互规范。

| 规范项 | 说明 |
|--------|------|
| **整体布局** | 左侧固定宽度导航栏（240px）+ 右侧自适应内容区（最小宽度 1200px） |
| **导航高亮** | 当前所在的一级菜单和二级菜单高亮，面包屑反映层级 |
| **面包屑** | 二级及更深页面顶部显示：一级菜单 > 二级菜单 > 当前页面 |
| **返回按钮** | 表单页/详情页左上角提供"← 返回列表" |
| **保存/取消** | 表单页底部固定或跟随内容末尾，主按钮在右（保存）、次按钮在左（取消） |
| **列表空状态** | 列表无数据时展示空状态插画 + "暂无{{数据名称}}" + 引导按钮（如"新增"） |
| **列表加载态** | 首次加载展示骨架屏（Skeleton），筛选/翻页展示 Spin |
| **列表错误态** | 接口异常展示错误提示 + 重试按钮 |
| **操作确认** | 删除、下线、批量操作等不可逆操作，必须弹出二次确认框 |
| **操作反馈** | 增删改操作成功弹出 Toast 提示（如"保存成功"），失败弹出错误提示 |
| **批量操作** | 列表左侧复选框，选中后出现批量操作栏，显示已选数量 |
| **分页** | 默认每页 20 条，支持切换 10/20/50/100，显示总条数和页码跳转 |
| **导出** | 导出操作为异步任务，点击后 Toast "导出任务已创建，请到导出记录下载" |
| **多 Tab** | 列表页内容较多时可用 Tab 切换不同数据视图（如"全部 / 已上线 / 已下线"） |
